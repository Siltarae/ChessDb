## 0. 현재 코드 상태와 이 작업의 위치

- **현재 상태 요약**: 분석 보드 레이아웃과 통계 API는 준비되었으나, 사용자에게 데이터를 표나 그래프로 보여줄 시각적 요소가 부족합니다.
- **이 작업의 책임**: 분석 보드 우측 패널에 API 결과를 렌더링하여, 사용자가 현재 포지션에서의 수 후보와 과거 성적을 한눈에 파악하게 합니다.

- **이번 작업에서 하지 않는 것**: `FEATURE-006` 완료에 연결된 후속 책임은 이번 태스크에서 함께 닫지 않는다.

- **경계 메모**:
  - 이번 태스크는 통계 패널 표시 및 시각화 범위만 닫고, 후속 태스크 또는 인접 Feature의 세부 구현은 여기서 함께 처리하지 않는다.

- **최종 상태**: 사용자가 보드에서 수를 두면 우측 패널이 즉시 업데이트되며, 가장 많이 두어진 수 순서대로 리스트가 나타나고 각 수별 승률이 막대 그래프로 표시됩니다.

- **이번 작업의 최소 결과물**:
  - `apps/web/src/widgets/statistics-panel/ui/statistics-panel.tsx`
  - `apps/web/src/features/statistics-view/model/use-position-statistics.ts`
  - `apps/web/src/pages/analysis-board-page.tsx`
- **성공 기준 (AC)**:
  - 보드 상태 변경 시 통계 데이터가 유기적으로(Loading -> Success) 업데이트된다.
  - 승률 바(Win Rate Bar)가 디자인 시안에 맞게 백색/회색/흑색 비율을 정확히 표현한다.
  - 통계 데이터가 없는 미탐색 포지션일 경우 "현재 저장소에 일치하는 기록이 없습니다" 메시지가 출력된다.

## 🛠️ 3. 상세 기술 사양

- **데이터 연동**:
  - `useAnalysisStore`에서 관리하는 `currentFen`이 바뀔 때마다 `TanStack Query`가 자동으로 통계 API를 재요청.
- **UI 컴포넌트**:
  - `StatsTable`: `shadcn/ui`의 Table을 사용하여 수(SAN), 대국 수(Count), 승률 표시.
  - `WinRateBar`: 가로형 스택 막대 그래프. 백승(White), 무승부(Gray), 흑승(Black) 영역의 너비를 승률 비율에 맞게 렌더링.
- **UX**:
  - 통계 리스트의 특정 수 클릭 시 해당 수를 실제로 분석 보드에 두어보는 '미리보기' 기능 연동.

- **핵심 조립/흐름 규칙**:
  - 보드 상태 변경 시 통계 데이터가 유기적으로(Loading -> Success) 업데이트된다.
  - 승률 바(Win Rate Bar)가 디자인 시안에 맞게 백색/회색/흑색 비율을 정확히 표현한다.
  - 통계 데이터가 없는 미탐색 포지션일 경우 "현재 저장소에 일치하는 기록이 없습니다" 메시지가 출력된다.

- **데이터 모델 해석**:
  - 입력은 API가 반환한 다음 수 후보와 결과 비율 통계입니다.
  - 출력은 화면에 렌더링할 통계 카드/차트/표 형태의 뷰 모델입니다.

- **외부 의존성**:
  - `react`
  - `@tanstack/react-query`
  - 차트 라이브러리 또는 shared chart wrapper

- **import/export 규칙**:
  - `../architecture/project-rules.md`의 named export, 상대 경로 최소화, `.js` 확장자 규칙을 따른다.

- **권장 네이밍**:
  - 공개 함수/타입 이름: 통계 패널 표시 및 시각화 책임이 드러나는 이름
  - 내부 helper 이름: 역할이 바로 드러나는 동사형 또는 조합형 이름
  - 핵심 변수명: 상태와 대상이 분명한 이름
  - 피해야 할 이름: data, item, obj, temp

- **이름별 사용 의도와 적용 시점**:
  - `usePositionStatistics`는 현재 포지션 기준 통계 fetch와 상태를 함께 노출할 때 사용합니다.
  - `statisticsItems`는 화면에 표시할 최종 후보 배열을 뜻합니다.

- **인수 이름 가이드**:
  - `repositoryId`, `positionKey`, `statisticsItems`

- **짧은 예시 골격**:

```ts
const { statisticsItems } = usePositionStatistics(repositoryId, positionKey);
```

- **필수 describe/it 목록**:
  - 최상위 describe: `통계 패널 표시 및 시각화`
  - 필수 it: 요구사항 문서의 완료 기준을 직접 검증하는 테스트

- **최소 테스트 개수**:
  - 최소 3개

- **반드시 포함할 실패 시나리오**:
  - API 응답 구조를 그대로 화면에 뿌려 표현 계층이 누수되는 경우
  - 포지션이 바뀌어도 이전 통계가 남아 있는 경우

## ⚖️ 4. 기술 제약 및 규칙

  - 전역 규칙을 반복 나열하지 말고, 이번 태스크에서 특히 강조해야 하는 제약만 짧게 적는다.

  - 현재 디렉토리 구조와 연관 설계 문서의 책임 경계를 유지한다.

- **범위 규칙**:
  - `FEATURE-006` 완료에 연결된 범위는 여기서 닫지 않는다.

- **헌법 정렬 규칙**:
  - `../architecture/project-rules.md`의 네이밍, import/export, 상태 규칙을 그대로 따른다.

- **문서화 규칙**:
