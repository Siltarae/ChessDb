## 0. 현재 코드 상태와 이 작업의 위치

- **현재 상태 요약**: 폰이 마지막 행에 도달해도 기물 타입이 변하지 않고 그대로 폰으로 남아있는 상태입니다.
- **이 작업의 책임**: 폰이 마지막 행에 도달했을 때 퀸, 룩, 비숍, 나이트 중 하나로 변신하는 프로모션 로직을 구현합니다.

## 🎯 1. 작업 목표

- **최종 상태**: 폰이 8행(White) 또는 1행(Black)에 도착할 때 사용자로부터 선택을 받거나 기본값(퀸)으로 기물 타입을 변경합니다.

## 🛠️ 3. 상세 기술 사양

- **판정 로직**: 폰의 `to` 인덱스가 `Math.floor(to / 8)` 기준으로 0 또는 7인지 확인.
- **인터페이스 확장**:
  - `Move` 인터페이스에 `promotion?: PieceType` 필드 활용.
  - `executeMove` 호출 시 `promotion` 값이 있으면 해당 타입으로 기물 생성.
- **필수 describe/it 목록**:
  - describe: `executeMove with Promotion`
    - it: `폰이 마지막 행 도달 시 지정된 기물(퀸 등)로 변신해야 한다`
    - it: `프로모션 정보가 누락된 경우의 기본값 처리 또는 에러 처리를 수행한다`

## ✅ 7. 완료 판정 체크리스트

- [ ] 폰이 보드 끝에 도달했을 때 프로모션이 정확히 발생한다.
- [ ] 퀸뿐만 아니라 나이트, 룩, 비숍으로의 선택 프로모션이 모두 가능하다.
- [ ] 프로모션 직후 보드에는 폰이 사라지고 새 기물이 나타남을 확인했다.
