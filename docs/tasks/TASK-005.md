## 0. 현재 코드 상태와 이 작업의 위치

- **현재 상태 요약**: `TASK-059`를 통해 `COLOR`, `PIECE_TYPE` 등의 상수가 마련되었습니다. 폰 로직(`TASK-004`)이 구현되었으나 나이트의 특수 이동(기물 뛰어넘기) 로직은 없습니다.
- **이 작업의 책임**: 나이트(Knight)의 8방향 'L'자 이동 가능 위치를 계산하는 독립적인 제너레이터를 구현합니다.
- **경계 메모**: 나이트는 경로 상의 기물을 무시합니다. 오직 목적지 칸에 아군 기물이 있는지 여부만 확인합니다.

## 🎯 1. 작업 목표

- **최종 상태**: 주어진 위치에 있는 나이트의 색상을 식별하고, 보드 경계를 벗어나지 않으며 아군 기물이 없는 모든 이동 가능한 좌표 배열을 반환합니다.
- **성공 기준 (AC)**:
  - 나이트 고유의 8가지 오프셋이 정확히 적용되어야 한다.
  - 보드 양쪽 끝에서 반대편으로 건너뛰는 '워프 현상'이 좌표 검증(File/Rank 차이)을 통해 완벽히 차단되어야 한다.
  - 목적지에 상대 기물이 있으면 결과에 포함하고(캡처), 아군 기물이 있으면 제외한다.

## 🛠️ 3. 상세 기술 사양

- **공개 함수 시그니처**:
  ```ts
  import { GameState, Square } from '../models/game-state.js';
  export function getKnightMoves(square: Square, state: GameState): Square[];
  ```
- **핵심 조립/흐름 규칙**:
  1. 현재 위치의 파일(File)과 랭크(Rank)를 계산한다. (`file = sq % 8`, `rank = floor(sq / 8)`)
  2. 나이트 오프셋: `[-17, -15, -10, -6, 6, 10, 15, 17]`을 순회한다.
  3. 각 타겟 인덱스에 대해:
     - 인덱스가 0~63 범위 내에 있는지 확인.
     - 타겟의 파일/랭크 차이가 (1, 2) 또는 (2, 1) 조합인지 확인하여 워프 방지.
     - 타겟 칸이 비어있거나 상대 색상 기물이 있는지 확인.
- **권장 네이밍**:
  - `KNIGHT_OFFSETS`: 오프셋 상수 배열.
  - `fileDiff`, `rankDiff`: 워프 검증용 변수.
- **필수 describe/it 목록**:
  - describe: `getKnightMoves`
    - it: `중앙(D4) 나이트는 8개의 합법 수 위치를 반환해야 한다`
    - it: `구석(A1) 나이트는 2개의 합법 수 위치만 반환해야 한다`
    - it: `목적지에 아군 기물이 있는 칸은 제외해야 한다`
    - it: `목적지에 상대 기물이 있는 칸은 결과에 포함해야 한다`
- **최소 테스트 개수**: 5개

## ⚖️ 4. 기술 제약 및 규칙

- **워프 차단**: 단순 인덱스 연산(`+10`)만으로는 보드 끝에서 끝으로 건너뛸 수 있으므로 반드시 좌표 기반 검증을 거칩니다.
- **불변성**: `state.board`를 읽기 전용으로만 사용합니다.
