## 0. 현재 코드 상태와 이 작업의 위치

- **현재 상태 요약**: 분석 보드에서 수를 두는 것은 가능하지만, 실수를 했거나 다른 경로를 보고 싶을 때 이전 포지션으로 돌아가는 수단이 없습니다.
- **이 작업의 책임**: 분석 중인 경로 상에서 앞뒤로 자유롭게 이동할 수 있는 Undo/Redo 기능을 구현합니다.

## 🛠️ 3. 상세 기술 사양

- **구현 전략**:
  - `useAnalysisStore` 내부에 `historyStack`과 `redoStack`을 관리.
  - **Undo**: `historyStack`에서 최상단 상태를 꺼내 `redoStack`에 넣고, 그 이전 상태를 현재 보드에 적용.
  - **Redo**: `redoStack`에서 상태를 꺼내 현재 보드에 적용하고 `historyStack`에 추가.
- **UI**:
  - 보드 하단에 컨트롤 바(버튼 세트: <<, <, >, >>) 배치.
  - 키보드 단축키(좌/우 방향키) 연동.

## ✅ 7. 완료 판정 체크리스트

- [ ] 수를 둔 후 '되돌리기' 클릭 시 정확히 이전 배치와 턴으로 복구된다.
- [ ] '다시하기'를 통해 취소했던 수를 다시 실행할 수 있다.
- [ ] 새로운 수를 두면 `redoStack`이 비워져 경로가 새롭게 갱신됨을 확인했다.
