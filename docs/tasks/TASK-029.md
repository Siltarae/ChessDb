**작업 상태**: 대기 중  
**선행 작업**: `[TASK-028]`  
**후속 작업**: `[TASK-033]`

## 0. 현재 코드 상태와 이 작업의 위치

- **현재 상태 요약**: 저장소 목록 조회는 가능하지만, 새로운 저장소를 추가할 수 있는 수단이 없습니다.
- **이 작업의 책임**: 사용자가 새로운 저장소 이름을 입력하고 서버에 생성 요청을 보내는 전체 인터랙션을 구현합니다.

## 🎯 1. 작업 목표

- **최종 상태**: 사용자가 버튼을 눌러 이름을 입력하고 확인을 누르면, 새로운 저장소가 목록에 즉시 추가됩니다.

## 🛠️ 3. 상세 기술 사양

- **UI**: 
  - `+ 새 저장소` 버튼 (목록 상단 또는 플로팅 버튼).
  - `shadcn/ui`의 Dialog(Modal) 컴포넌트를 사용하여 이름 입력 필드 노출.
- **검증**: 이름은 최소 1자 이상이어야 하며, 공백만으로는 생성 불가.
- **통신**: `POST /api/repositories` 엔드포인트 호출 및 `queryClient.invalidateQueries`를 통한 목록 갱신.

## ✅ 4. 완료 판정 체크리스트

- [ ] 저장소 생성 모달이 정상적으로 열리고 닫힌다.
- [ ] 유효한 이름을 입력했을 때 서버에 저장이 성공하고 목록이 새로고침된다.
- [ ] 중복된 이름이나 빈 이름에 대한 처리가 적절히 이루어진다(선택 사항).
