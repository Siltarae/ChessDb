## 0. 현재 코드 상태와 이 작업의 위치

- **현재 상태 요약**: 저장소 목록 조회는 가능하지만, 새로운 저장소를 추가할 수 있는 수단이 없습니다.
- **이 작업의 책임**: 사용자가 새로운 저장소 이름을 입력하고 서버에 생성 요청을 보내는 전체 인터랙션(모달 오픈 -> 입력 -> 전송 -> 갱신)을 구현합니다.

## 🎯 1. 작업 목표

- **최종 상태**: 사용자가 '+ 새 저장소' 버튼을 누르고 이름을 입력하면, 목록에 즉시 새로운 저장소 카드가 추가됩니다.

## 🛠️ 3. 상세 기술 사양

- **UI 흐름**:
  - `shadcn/ui`의 `Dialog` 컴포넌트를 활용한 입력 모달.
  - `React Hook Form` + `Zod`를 이용한 이름 필드 검증 (최소 1자, 최대 50자).
- **검증 규칙**:
  - 앞뒤 공백 제거 후 1자 이상 필수.
  - 중복된 이름은 서버 에러 메시지에 따라 적절히 사용자에게 알림.
- **API 연동**:
  - `POST /api/repositories` 엔드포인트 호출.
  - 성공 시 `queryClient.invalidateQueries(['repositories'])` 호출로 목록 동기화.

## ✅ 7. 완료 판정 체크리스트

- [ ] 생성 중 로딩 상태(Button disabled 등) 처리가 되어 있다.
