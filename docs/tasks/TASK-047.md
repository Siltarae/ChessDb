## 0. 현재 코드 상태와 이 작업의 위치

- **현재 상태 요약**: 기보 목록과 상세 보기가 가능하지만, 잘못 기록된 기보를 제거할 수단이 없습니다.
- **이 작업의 책임**: 기보를 영구적으로 삭제하는 기능을 구현하며, 실수로 인한 삭제를 방지하기 위한 확인 절차를 포함합니다.

## 🛠️ 3. 상세 기술 사양

- **UI**:
  - 기보 상세 화면의 헤더 또는 목록의 각 항목 메뉴에 삭제 버튼 배치.
  - `shadcn/ui`의 `AlertDialog` 사용: "이 기보를 영구적으로 삭제하시겠습니까? 삭제된 기보는 복구할 수 없습니다."
- **API 연동**:
  - `DELETE /api/games/:id` 호출.
  - 성공 시 `toast.success` 표시 후 목록 페이지로 리다이렉트.
- **백엔드 (API)**:
  - `GamesService.remove` 메서드 구현.
  - DB의 `deletedAt` 컬럼을 활용한 Soft Delete 권장 (프로젝트 규칙 준수).

## ✅ 7. 완료 판정 체크리스트

- [ ] 삭제 버튼 클릭 시 명시적인 확인 모달이 나타난다.
- [ ] 삭제 완료 후 해당 기보가 목록에서 즉시 제거되고 페이지가 이동한다.
- [ ] Soft Delete가 적용되어 실제 DB 레코드는 유지되면서 사용자에게만 안 보이게 처리됨을 확인했다.
