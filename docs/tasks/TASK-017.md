## 0. 현재 코드 상태와 이 작업의 위치

- **현재 상태 요약**: 기물은 움직이지만 지금까지 어떤 수들이 두어졌는지에 대한 이력이 화면에 남지 않습니다.
- **이 작업의 책임**: 게임 시작부터 현재까지의 모든 착수 내역을 배열로 관리하고, 우측 패널에 표준 기보 형식으로 나열합니다.

## 🎯 1. 작업 목표

- **최종 상태**: 사용자가 수를 둘 때마다 우측 `MoveList` 영역에 `1. e4 e5 2. Nf3 ...` 형태의 기보가 실시간으로 쌓입니다.

## 🛠️ 3. 상세 기술 사양

- **데이터 구조**: `history: MoveHistoryItem[]` (from `TASK-063`).
- **UI 컴포넌트**:
  - `MoveListContainer`: 전체 수순을 감싸는 스크롤 영역.
  - `MoveItem`: 개별 수(Ply)를 렌더링하며, 현재 보고 있는 수를 강조.
- **렌더링 규칙**:
  - 2개 수(백, 흑)를 묶어 하나의 행(Move Number)으로 표시.
  - 수순이 많아질 경우 자동으로 최하단으로 스크롤 이동 (`auto-scroll`).

## ✅ 7. 완료 판정 체크리스트

- [ ] 모든 착수가 순서대로 정확한 SAN 표기법과 함께 목록에 추가된다.
- [ ] 수 번호(1, 2, 3...)가 백의 수 앞에 정확히 붙는다.
- [ ] 목록 클릭 시 해당 시점의 보드를 보여줄 준비가 되어 있다 (데이터 링크 확인).
