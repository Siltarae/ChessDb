## 0. 현재 코드 상태와 이 작업의 위치

- **현재 상태 요약**: 프론트엔드와 백엔드가 분리되어 있어, 통신 데이터의 타입 일관성을 보장하기 어려운 상태입니다.
- **이 작업의 책임**: `packages/shared`에서 정의한 Zod 스키마를 백엔드의 요청 데이터 검증(ValidationPipe)과 프론트엔드의 타입 정의에 동시에 사용하도록 구조를 연결합니다.

## 🛠️ 3. 상세 기술 사양

- **DTO 위치**: `packages/shared/src/dto/`
- **사용 방식**:
  - 백엔드: `nestjs-zod` 등을 활용하여 Zod 스키마를 NestJS `Pipe` 및 `Swagger`와 연동.
  - 프론트엔드: `z.infer<typeof Schema>`를 통해 요청/응답 타입을 자동 생성하여 사용.
- **샘플 DTO**: `CreateGameDto`, `UpdateCommentDto` 등.

- [ ] 잘못된 형식의 API 요청 시 백엔드에서 Zod 검증에 의해 400 에러가 정상 발생한다.
