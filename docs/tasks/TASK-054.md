## 0. 현재 코드 상태와 이 작업의 위치

- **현재 상태 요약**: DB에 기보들이 저장되어 있으나, 특정 포지션(FEN)을 기준으로 다른 기보들에서 어떤 수가 두어졌는지 찾아내는 쿼리 로직이 없습니다.
- **이 작업의 책임**: 특정 저장소 내의 모든 기보를 대상으로, 현재 분석 중인 포지션과 일치하는 시점을 찾고 그 다음 수(Move)와 결과(Result)를 집계하는 API를 구현합니다.

## 🎯 1. 작업 목표

- **최종 상태**: `GET /api/repositories/:id/stats?fen=...` 요청 시 해당 포지션의 통계 데이터(수 후보, 승률 등)를 반환합니다.

## 🛠️ 3. 상세 기술 사양

- **데이터베이스 쿼리 (Prisma/SQL)**:
  - `Game` 테이블과 연관된 `Move` 테이블에서 특정 `fen` 값과 일치하는 레코드들을 필터링.
  - 일치하는 각 수(Move)의 다음 수(`next_move_san`)별로 그룹화(`groupBy`)하여 카운트 집계.
  - 해당 게임들의 `result`('1-0', '1/2-1/2', '0-1') 비율을 함께 계산.
- **성능 고려**: 기보가 많아질 경우를 대비해 `Move.fen` 컬럼에 인덱스를 적용합니다.

## ✅ 4. 완료 판정 체크리스트

- [ ] 특정 포지션에 대해 저장소 내 모든 기보의 통계가 정확히 합산된다.
- [ ] 백승, 무승부, 흑승 비율이 백분율(%)로 계산되어 반환된다.
- [ ] 결과 데이터가 없는 포지션일 경우 빈 목록이 에러 없이 반환된다.
