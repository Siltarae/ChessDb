## 0. 현재 코드 상태와 이 작업의 위치

- **현재 상태 요약**: 데이터베이스에 기보들이 저장되어 있으나, 특정 포지션(FEN)을 기준으로 다른 기보들에서 어떤 수가 두어졌는지 찾아내는 집계 로직이 없습니다.
- **이 작업의 책임**: 특정 저장소 내의 모든 기보 데이터를 분석하여, 현재 포지션과 일치하는 시점의 다음 수(Moves) 통계와 해당 게임들의 결과 비율을 계산하는 API를 구축합니다.

## 🎯 1. 작업 목표

- **최종 상태**: 사용자가 분석 보드에서 특정 모양을 만들면, "내 저장소 내의 다른 게임들에서는 이 상황에서 e4가 10번 두어졌고 백의 승률이 60%였다"는 정보를 API를 통해 얻을 수 있습니다.

## 🛠️ 3. 상세 기술 사양

- **백엔드 (API)**:
  - `GET /api/repositories/:repoId/stats?fen={encodedFen}` 엔드포인트 구현.
  - **데이터베이스 쿼리 (Prisma Raw Query 또는 복합 쿼리)**:
    1. 특정 `repoId`에 속한 모든 `Game`의 `Move` 데이터 필터링.
    2. 입력받은 `fen` 문자열과 일치하는 `Move` 레코드 식별.
    3. 해당 레코드들의 바로 다음 수(`nextMove`) 정보를 그룹화하여 빈도수 계산.
    4. 연관된 `Game.result`('1-0', '1/2-1/2', '0-1') 정보를 합산하여 승률 통계 도출.
- **성능 최적화**:
  - `Move` 테이블의 `fen` 컬럼에 인덱스 필수 적용.
  - `fen` 문자열 비교 시 공백이나 앙파상 권리 등 정규화된 형태를 유지하도록 주의.

## ✅ 7. 완료 판정 체크리스트

- [ ] 특정 포지션에 대해 저장소 내 모든 기보의 통계가 정확히 합산되어 반환된다.
- [ ] 백승, 무승부, 흑승 비율이 숫자(Percentage)로 정확히 계산된다.
- [ ] 대량의 기보 데이터에서도 인덱스를 통해 빠르게 통계를 조회할 수 있다.
