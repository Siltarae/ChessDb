# 📋 개별 작업 지침서: 정식 기보 코멘트 및 어노테이션 수정 (TASK-043, TASK-044, TASK-045, TASK-046)

## 0. 현재 코드 상태와 이 작업의 위치

- **현재 상태 요약**: 기보의 수순은 볼 수 있으나, 함께 저장된 코멘트를 확인하거나 사후에 수정하는 기능이 없습니다.
- **이 작업의 책임**: 저장된 기보의 반수별 코멘트와 평가 기호를 표시하고, 사용자가 이를 수정하면 즉시 서버에 자동 저장되도록 구현합니다.

## 🎯 1. 작업 목표

- **최종 상태**: 사용자가 복기 중에 새로운 깨달음을 얻어 코멘트를 수정하면, 별도의 저장 버튼 없이도 서버에 반영됩니다.

## 🛠️ 3. 상세 기술 사양

- **수정 제약**: 정식 저장된 기보의 **수순(Moves)**은 수정할 수 없으나, **메타데이터(Comments, Annotations)**는 자유롭게 수정 가능합니다.
- **자동 저장**: `onBlur` 또는 디바운스된 `onChange` 시점에 `PATCH /api/games/:id/metadata` API 호출.
- **UI**: 입력 화면(`FEATURE-003`)에서 사용한 코멘트 입력 컴포넌트를 재사용하되, 수정 시 로딩 상태 표시 추가.

## ✅ 4. 완료 판정 체크리스트

- [ ] 저장되어 있던 코멘트와 평가 기호가 각 수순에 맞게 정확히 표시된다.
- [ ] 코멘트 수정 후 페이지를 나갔다 들어와도 수정된 내용이 유지된다 (서버 저장 확인).
- [ ] 수정 중 네트워크 오류 발생 시 적절한 경고가 표시된다.
