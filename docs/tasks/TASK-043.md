## 0. 현재 코드 상태와 이 작업의 위치

- **현재 상태 요약**: 기보의 수순 탐색은 가능하나, 저장된 코멘트를 확인하거나 복기 중에 새로운 코멘트를 추가/수정하는 기능이 없습니다.
- **이 작업의 책임**: 저장된 기보의 반수별 코멘트와 평가 기호를 표시하고, 수정 시 서버에 즉시 반영하는 자동 저장(Auto-sync) 로직을 구현합니다.

## 🛠️ 3. 상세 기술 사양

- **수정 제약**: 수순(Moves) 자체는 불변이나, 메타데이터(Comments, Annotations)는 수정 가능한 구조로 설계합니다.
- **자동 저장 로직**:
  - `CommentEditor`에서 포커스를 잃을 때(`onBlur`) 또는 내용 변경 후 일정 시간(1s) 대기 시 서버 요청.
  - `PATCH /api/games/:id/history/:index` 엔드포인트 구현 (특정 수순의 코멘트 업데이트).
- **UI**:
  - 입력 화면에서 사용한 `CommentEditor`를 상세 화면용으로 재사용.
  - 수정 중 '저장 중...'과 같은 로딩 상태를 작게 표시하여 사용자에게 알림.

## ✅ 7. 완료 판정 체크리스트

- [ ] 기존에 저장된 코멘트와 평가 기호가 각 수순에 맞게 정확히 로드된다.
- [ ] 수정 후 새로고침하거나 다른 페이지를 다녀와도 수정 내용이 유지된다.
- [ ] 서버 저장 실패 시 사용자에게 경고 메시지를 보여주고 이전 상태로 롤백하거나 재시도를 유도한다.
