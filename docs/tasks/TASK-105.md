## 0. 현재 코드 상태와 이 작업의 위치

- **현재 상태 요약**: 저장소 기본 보드뷰는 보드 작업 상태를 가질 수 있지만, 사용자가 변경한 로컬 상태를 원본으로 되돌리는 규칙은 아직 없습니다.
- **이 작업의 책임**: 기존 기보가 있는 경우 마지막 저장 상태로 복원하고, 기존 기보가 없는 경우 표준 시작 포지션으로 초기화하는 규칙을 구현합니다.
- **이번 작업에서 하지 않는 것**: 기존 기보 업데이트 저장은 `[TASK-104]`에서 다루며, 새 기보 작성 진입은 `[TASK-035]`에서 다룹니다.
- **경계 메모**:
  - 복원은 로컬 작업 상태만 바꾸며 서버의 정식 저장 기보를 수정하거나 삭제하지 않습니다.

## 🎯 1. 작업 목표

- **최종 상태**: 사용자가 저장소 보드에서 현재 변경 사항을 버리고 원본 기준 상태로 돌아갈 수 있습니다.
- **이번 작업의 최소 결과물**:
  - `apps/web/src/features/repository-board-restore/**`
  - `apps/web/src/pages/repository-home-page.tsx`
  - 필요 시 `apps/web/src/entities/game/**`
- **성공 기준 (AC)**:
  - 기존 `gameId`가 있으면 마지막 저장된 기보 상태로 복원된다.
  - 기존 `gameId`가 없으면 표준 시작 포지션과 빈 수순으로 초기화된다.
  - 복원 전 변경 사항 손실 확인 흐름을 제공한다.
  - 복원은 서버 저장 API를 호출하지 않는다.

## 📂 2. 대상 아티팩트

- **신규 생성**:
  - `apps/web/src/features/repository-board-restore/model/use-restore-repository-board.ts`
  - `apps/web/src/features/repository-board-restore/ui/restore-repository-board-button.tsx`
- **수정 대상**:
  - `apps/web/src/pages/repository-home-page.tsx`
  - 필요 시 현재 보드/수순 store 관련 selector와 action
- **이번 작업에서 수정하지 않음**:
  - `apps/api/src/modules/games/**`
  - 기존 기보 업데이트 저장 API
  - 저장소 목록 UI

## 🛠️ 3. 상세 기술 사양

- 복원 대상은 현재 저장소 보드의 로컬 작업 상태입니다.
- `gameId`가 있으면 서버 또는 캐시의 마지막 저장 상태를 원본으로 사용합니다.
- `gameId`가 없으면 표준 시작 포지션, 빈 수순, 빈 메타데이터를 원본으로 사용합니다.
- 변경 사항이 있을 때는 확인 다이얼로그를 거친 뒤 복원합니다.
- 복원 액션은 `POST /api/games`나 `PATCH /api/games/:gameId`를 호출하지 않습니다.

## ⚖️ 4. 기술 제약 및 규칙

- 복원은 영구 저장된 데이터를 변경하지 않습니다.
- `초기화`와 `원본으로 복원` 용어 중 실제 UI 문구는 구현 전에 하나로 확정합니다.
- 기존 기보 원본 상태와 로컬 작업 상태를 혼동하지 않습니다.

## 🧪 5. 검증 시나리오 및 단언

1. **정상 시나리오: 기존 기보 복원**
   - `gameId`가 있는 상태에서 보드/수순을 변경한다.
   - 복원하면 마지막 저장 상태로 돌아간다.

2. **정상 시나리오: 빈 작업 상태 초기화**
   - `gameId`가 없는 상태에서 보드/수순을 변경한다.
   - 복원하면 표준 시작 포지션과 빈 수순으로 돌아간다.

3. **실패 시나리오: 서버 저장 호출**
   - 복원 버튼 클릭 시 저장 API가 호출된다.
   - 테스트가 실패해야 한다.

## 🚀 6. 권장 작업 순서

1. 원본 상태와 로컬 작업 상태를 구분하는 입력 모델을 확정합니다.
2. 복원 hook을 작성합니다.
3. 복원 버튼과 확인 다이얼로그를 연결합니다.
4. 기존 기보 상태와 빈 작업 상태 테스트를 각각 추가합니다.

- **검증 실행**:
  - `pnpm --filter @chess-db/web test`
  - `pnpm --filter @chess-db/web build`
