## 0. 현재 코드 상태와 이 작업의 위치

- **현재 상태 요약**: 기물을 선택하고 갈 수 있는 곳을 볼 수는 있으나, 실제로 말을 옮겨 게임을 진행할 수는 없습니다.
- **이 작업의 책임**: 하이라이트된 합법 수 칸을 클릭했을 때 실제로 상태를 전이시키고 보드를 갱신합니다.

- **이번 작업에서 하지 않는 것**: `[TASK-017]` (수순 기록)에 연결된 후속 책임은 이번 태스크에서 함께 닫지 않는다.

- **경계 메모**:
  - 이번 태스크는 합법 수 착수와 보드 갱신 범위만 닫고, 후속 태스크 또는 인접 Feature의 세부 구현은 여기서 함께 처리하지 않는다.
  - 착수 완료 후 마지막 수 표시를 남기고 선택 칸과 합법 수 하이라이트는 정리한다.

  - 캡처 상황 시 상대 기물이 보드에서 사라지고 내 기물이 그 자리를 차지한다.

## 🛠️ 3. 상세 기술 사양

- **인터랙션 로직**:
  - `selectedSquare`가 있는 상태에서 '합법 수'로 표시된 `targetSquare` 클릭.
  - `executeMove(currentGameState, { from: selectedSquare, to: targetSquare })` 호출.
  - 반환된 새로운 `GameState`로 Zustand 스토어 업데이트.
  - `selectedSquare`를 `null`로 초기화하여 선택 해제.
- **핵심 조립/흐름 규칙**:
  - 기물이 시작점에서 목표점으로 정확히 이동한다.
  - 착수 완료 후 즉시 반대편 색상의 턴으로 전환된다.
  - 캡처 상황 시 상대 기물이 보드에서 사라지고 내 기물이 그 자리를 차지한다.
  - 마지막 수는 이전/도착 칸이 구분되는 방식으로 표시한다.

- **데이터 모델 해석**:
  - 입력은 현재 보드 상태, 선택한 출발 칸/도착 칸, 실행할 합법 수입니다.
  - 출력은 갱신된 보드 상태와 턴, 선택 해제 결과입니다.

- **외부 의존성**:
  - `react`
  - `zustand`
  - `@chess-db/shared` `executeMove` 또는 동등 엔진

- **import/export 규칙**:
  - `../architecture/project-rules.md`의 named export, 상대 경로 최소화, `.js` 확장자 규칙을 따른다.

- **권장 네이밍**:
  - 공개 함수/타입 이름: 합법 수 착수와 보드 갱신 책임이 드러나는 이름
  - 내부 helper 이름: 역할이 바로 드러나는 동사형 또는 조합형 이름
  - 핵심 변수명: 상태와 대상이 분명한 이름
  - 피해야 할 이름: data, item, obj, temp

- **이름별 사용 의도와 적용 시점**:
  - `useMakeMove`는 보드 입력과 상태 전이를 연결하는 훅 이름으로 사용합니다.
  - `pendingMove`는 아직 확정되지 않은 사용자 선택 상태를 뜻할 때 사용합니다.

- **인수 이름 가이드**:
  - `fromSquare`, `toSquare`, `nextState`

- **짧은 예시 골격**:

```ts
const { makeMove } = useMakeMove();
makeMove(toSquare);
```

- **필수 describe/it 목록**:
  - 최상위 describe: `합법 수 착수와 보드 갱신`
  - 필수 it: 요구사항 문서의 완료 기준을 직접 검증하는 테스트

- **최소 테스트 개수**:
  - 최소 3개

- **반드시 포함할 실패 시나리오**:
  - 합법 수 검증 없이 임의 칸 이동을 허용하는 경우
  - 착수 후 선택 상태를 정리하지 않아 다음 입력과 충돌하는 경우

- **작성 원칙**:

  - `[TASK-017]` (수순 기록)에 연결된 범위는 여기서 닫지 않는다.

- **헌법 정렬 규칙**:
