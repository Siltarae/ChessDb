## 0. 현재 코드 상태와 이 작업의 위치

- **현재 상태 요약**: 폰의 일반 이동과 캡처는 구현되었습니다. 상대 폰이 2칸 전진했을 때만 발생하는 특수 캡처 규칙인 앙파상 로직이 누락되어 있습니다.
- **이 작업의 책임**: `enPassantSquare` 데이터를 활용하여 앙파상 캡처 가능 여부를 판정하고, 실행 시 배후의 폰을 제거합니다.

- **이번 작업에서 하지 않는 것**: `[TASK-014]` (프로모션)에 연결된 후속 책임은 이번 태스크에서 함께 닫지 않는다.

- **경계 메모**:
  - 이번 태스크는 앙파상(En Passant) 처리 범위만 닫고, 후속 태스크 또는 인접 Feature의 세부 구현은 여기서 함께 처리하지 않는다.

## 🛠️ 3. 상세 기술 사양

- **판정 규칙**:
  - `state.enPassantSquare`가 `null`이 아님.
  - 내 폰이 해당 칸으로 대각선 1칸 이동하여 캡처할 수 있는 위치에 있음.
- **실행 규칙 (특수 처리)**:
  - 폰은 `enPassantSquare` 인덱스로 이동한다.
  - 하지만 실제로 잡히는 상대 폰은 한 칸 뒤(또는 앞) 인덱스에 있으므로, 해당 위치의 기물을 `null`로 만든다.
- **필수 describe/it 목록**:
  - describe: `getEnPassantMoves`
    - it: `앙파상 타겟이 설정된 경우 대각선 캡처 수를 반환해야 한다`
    - it: `앙파상 실행 시 상대 폰이 있던 원래 위치의 기물이 제거되어야 한다`
    - it: `앙파상 기회는 해당 턴에만 유효하고 다음 턴에는 소멸되어야 한다`

- **핵심 조립/흐름 규칙**:
  - 앙파상 캡처 수가 정확히 계산되어 반환된다.
  - 캡처 실행 시 피격 폰이 보드에서 정확히 제거됨을 확인했다.
  - 앙파상 유효 기간(1턴)이 정확히 관리된다.

- **데이터 모델 해석**:
  - 앙파상은 직전 수의 2칸 전진 정보와 현재 폰 위치, 캡처 대상 칸을 함께 다룹니다.
  - 출력은 앙파상 가능 수 또는 상태 전이용 특수 수 정보입니다.

- **외부 의존성**:
  - shared 내부 `GameState`, `Move` 타입
  - `vitest`

- **import/export 규칙**:
  - `../architecture/project-rules.md`의 named export, 상대 경로 최소화, `.js` 확장자 규칙을 따른다.

- **권장 네이밍**:
  - 공개 함수/타입 이름: 앙파상(En Passant) 처리 책임이 드러나는 이름
  - 내부 helper 이름: 역할이 바로 드러나는 동사형 또는 조합형 이름
  - 핵심 변수명: 상태와 대상이 분명한 이름
  - 피해야 할 이름: data, item, obj, temp

- **이름별 사용 의도와 적용 시점**:
  - `getEnPassantMoves`는 현재 가능한 앙파상 수를 찾을 때 사용합니다.
  - `enPassantTarget`은 상태에 저장된 캡처 가능 칸 이름으로 유지합니다.

- **인수 이름 가이드**:
  - `state`, `square`, `enPassantTarget`

- **짧은 예시 골격**:

```ts
const enPassantMoves = getEnPassantMoves(state, square);
```

- **최소 테스트 개수**:
  - 최소 3개

- **반드시 포함할 실패 시나리오**:
  - 직전 수 조건 없이 앙파상을 허용하는 경우
  - 캡처된 폰 제거를 상태 전이에서 놓치는 경우
