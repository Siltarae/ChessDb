## 0. 현재 코드 상태와 이 작업의 위치

- **현재 상태 요약**: 폰의 일반 이동과 캡처는 구현되었습니다. 상대 폰이 2칸 전진했을 때만 발생하는 특수 캡처 규칙인 앙파상 로직이 누락되어 있습니다.
- **이 작업의 책임**: `enPassantSquare` 데이터를 활용하여 앙파상 캡처 가능 여부를 판정하고, 실행 시 배후의 폰을 제거합니다.

## 🎯 1. 작업 목표

- **최종 상태**: 앙파상 조건 만족 시 대각선 빈 칸으로의 이동을 합법 수로 제안하고, 착수 시 상대 폰을 보드에서 정확히 제거합니다.

## 🛠️ 3. 상세 기술 사양

- **판정 규칙**:
  - `state.enPassantSquare`가 `null`이 아님.
  - 내 폰이 해당 칸으로 대각선 1칸 이동하여 캡처할 수 있는 위치에 있음.
- **실행 규칙 (특수 처리)**:
  - 폰은 `enPassantSquare` 인덱스로 이동한다.
  - 하지만 실제로 잡히는 상대 폰은 한 칸 뒤(또는 앞) 인덱스에 있으므로, 해당 위치의 기물을 `null`로 만든다.
- **필수 describe/it 목록**:
  - describe: `getEnPassantMoves`
    - it: `앙파상 타겟이 설정된 경우 대각선 캡처 수를 반환해야 한다`
    - it: `앙파상 실행 시 상대 폰이 있던 원래 위치의 기물이 제거되어야 한다`
    - it: `앙파상 기회는 해당 턴에만 유효하고 다음 턴에는 소멸되어야 한다`

## ✅ 7. 완료 판정 체크리스트

- [ ] 앙파상 캡처 수가 정확히 계산되어 반환된다.
- [ ] 캡처 실행 시 피격 폰이 보드에서 정확히 제거됨을 확인했다.
- [ ] 앙파상 유효 기간(1턴)이 정확히 관리된다.
