## 0. 현재 코드 상태와 이 작업의 위치

- **현재 상태 요약**: 저장소 목록 조회가 가능하지만, 잘못 만들었거나 더 이상 필요 없는 저장소를 제거할 방법이 없습니다.
- **이 작업의 책임**: 저장소를 삭제할 때 발생할 수 있는 데이터 손실(하위 기보 모두 삭제)을 사용자에게 알리고, 안전하게 삭제 요청을 처리합니다.

## 🎯 1. 작업 목표

- **최종 상태**: 삭제 버튼 클릭 시 강력한 경고 메시지가 노출되며, 확인 후 서버에서 데이터가 영구적으로 제거됩니다.

## 🛠️ 3. 상세 기술 사양

- **UI**:
  - 저장소 카드 내 또는 상세 페이지 관리 메뉴에 삭제 버튼 배치.
  - `shadcn/ui`의 `AlertDialog` 사용: "정말로 삭제하시겠습니까? 이 작업은 되돌릴 수 없으며 포함된 모든 기보가 사라집니다."
- **API 연동**:
  - `DELETE /api/repositories/:repoId` 호출.
  - 성공 시 목록 페이지로 이동하거나 목록을 갱신.
- **백엔드 (API)**:
  - `Prisma`의 `onDelete: Cascade` 설정을 확인하거나 서비스 단에서 연관된 기보들을 먼저 처리.

## ✅ 7. 완료 판정 체크리스트

- [ ] 삭제 전 반드시 사용자의 명시적인 확인(Confirm) 절차를 거친다.
- [ ] 삭제 성공 후 목록에서 해당 데이터가 즉시 사라짐을 확인했다.
- [ ] 삭제된 저장소의 URL로 직접 접근 시 404 에러 또는 목록으로 리다이렉트 처리된다.
