# 📋 개별 작업 지침서: 정식 저장 확정 (TASK-025)

**작업 상태**: 대기 중  
**선행 작업**: `[TASK-090]`, `[TASK-022]`  
**후속 작업**: `FEATURE-003` 완료

---

## 0. 현재 코드 상태와 이 작업의 위치

- **현재 상태 요약**: 로컬 저장은 완벽하나, 다른 기기에서 보거나 영구 보존하기 위해 서버(DB)에 데이터를 전송하는 로직이 없습니다.
- **이 작업의 책임**: 현재의 모든 기보 데이터(수순, 메타데이터)를 백엔드 API로 전송하여 PostgreSQL에 영구 저장합니다.

## 🎯 1. 작업 목표

- **최종 상태**: '정식 저장' 버튼 클릭 시 데이터가 서버에 기록되며, 성공 후 로컬의 초안은 폐기되거나 완료 상태로 전환됩니다.

## 🛠️ 3. 상세 기술 사양

- **프론트엔드**: `useMutation` (TanStack Query)을 사용하여 `POST /api/games` 요청.
- **백엔드**: 
  - `GamesController` 및 `GamesService` 구현.
  - `PrismaService`를 사용하여 `Game` 및 `Move` 테이블(또는 JSONB 필드)에 데이터 저장.
- **성공 후 처리**: 저장된 기보의 상세 화면(FEATURE-005)으로 리다이렉트.

## ✅ 4. 완료 판정 체크리스트

- [ ] 서버에 데이터가 에러 없이 전송되고 DB에 기록된다.
- [ ] 저장 성공 후 로컬 초안이 성공적으로 정리된다.
- [ ] 에러 발생 시 사용자에게 적절한 피드백(토스트 메시지 등)이 제공된다.
