## 0. 현재 코드 상태와 이 작업의 위치

- **현재 상태 요약**: 보드 표시, 선택/하이라이트, 착수 실행은 컴포넌트와 훅 테스트로 검증되지만, 실제 브라우저에서 사용자가 기보 입력 화면을 조작하는 전체 흐름 검증은 없습니다.
- **이 작업의 책임**: Playwright로 기보 입력 화면의 가장 작은 핵심 사용자 흐름을 스모크 테스트로 고정합니다.
- **이번 작업에서 하지 않는 것**: 체크메이트, 프로모션, 체크 표시, 게임 종료 표시, SAN, 수순 목록, 저장, 코멘트, 평가, 기보 메타데이터 입력은 검증하지 않습니다.
- **경계 메모**:
  - `[TASK-016]` 자체 완료 판정에는 E2E를 소급 포함하지 않습니다.
  - 이 Task는 `FEATURE-002` 웹 UI 흐름을 별도 E2E 안전망으로 보강하는 후속 검증입니다.

## 🎯 1. 작업 목표

- **최종 상태**: 사용자가 브라우저에서 기보 입력 화면에 들어가 기물을 선택하고 합법 수로 착수했을 때, 보드와 턴 표시가 갱신되는 핵심 흐름을 E2E로 확인합니다.
- **이번 작업의 최소 결과물**:
  - `apps/web/e2e/notation-input-smoke.spec.ts`
  - 필요 시 E2E 셀렉터 보강을 위한 접근성 이름 또는 `data-testid`
- **성공 기준 (AC)**:
  - 기보 입력 화면이 정상 로딩된다.
  - 사용자가 `e2` 백 폰을 선택하면 합법 수 후보가 화면에 드러난다.
  - `e4`를 클릭하면 폰이 `e4`로 이동한다.
  - 착수 후 현재 턴 표시가 흑으로 바뀐다.
  - 불법 칸 클릭은 보드 상태를 바꾸지 않는다.

  - 테스트를 위해 사용자에게 보이지 않는 임의 기능을 추가하지 않는다.

## 🛠️ 3. 상세 기술 사양

- 테스트는 `TASK-016` 범위의 첫 착수 흐름만 검증한다.
- 첫 정상 착수 시나리오는 `e2 -> e4`로 고정한다.
- 보드 칸 선택은 접근성 이름 또는 안정적인 테스트 식별자를 사용한다.
- 기물 존재 여부는 이미지 파일명보다 보드 칸의 의미 있는 상태를 우선 확인한다.
- 턴 표시는 사용자가 읽을 수 있는 UI 텍스트 또는 접근성 이름으로 확인한다.
- SAN, 수순 목록, 프로모션, 체크, 게임 종료 표시는 이 Task의 필수 단언으로 넣지 않는다.

## 🧪 4. 검증 시나리오 및 단언

1. **정상 시나리오: 첫 착수 스모크**
   - 기보 입력 화면에 접속한다.
   - `e2` 백 폰을 선택한다.
   - 합법 수 후보 중 `e4`를 클릭한다.
   - `e2`는 비고 `e4`에 백 폰이 표시된다.
   - 현재 턴은 흑으로 바뀐다.

2. **실패 시나리오: 불법 칸 클릭**
   - `e2` 백 폰을 선택한다.
   - 합법 수가 아닌 칸을 클릭한다.
   - 보드 상태와 현재 턴은 바뀌지 않는다.

3. **경계 시나리오: 재선택**
   - 기물을 선택한 뒤 다른 내 기물을 선택한다.
   - 이전 합법 수 표시는 정리되고 새 선택 기준의 합법 수만 표시된다.

4. **범위 침범 차단**
   - 수순 목록, SAN 문자열, 저장 성공 여부는 이 Task의 필수 단언으로 넣지 않는다.
