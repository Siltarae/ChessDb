## 0. 현재 코드 상태와 이 작업의 위치

- **현재 상태 요약**: 기보 입력 페이지는 존재하지만, 사용자가 처음 진입하여 저장소를 선택할 수 있는 메인 '대시보드' 또는 '저장소 목록' 페이지가 없습니다.
- **이 작업의 책임**: 사용자가 보유한 모든 저장소를 조회하여 카드 또는 리스트 형태로 보여주는 메인 페이지를 구축합니다.

## 🎯 1. 작업 목표

- **최종 상태**: 사용자가 앱 진입 시 자신이 만든 저장소들의 목록을 생성 순서대로 확인합니다.
- **핵심 산출물**: `RepositoryListPage.tsx`, `RepositoryCard.tsx`, `GET /api/repositories` API 엔드포인트.

## 🛠️ 3. 상세 기술 사양

- **데이터 모델**: 
  - `Repository`: `{ id: string, name: string, createdAt: string, gameCount: number }`
- **정렬 로직**: 
  - 백엔드: `Prisma` 조회 시 `orderBy: { createdAt: 'asc' }` 적용.
- **UI**: `shadcn/ui`의 Card 컴포넌트를 사용하여 저장소 이름과 생성일, 포함된 기보 수를 표시.

## ✅ 4. 완료 판정 체크리스트

- [ ] 저장소 목록이 화면에 그리드 또는 리스트 형태로 정상 출력된다.
- [ ] 생성된 순서(과거순)대로 목록이 정렬되어 표시된다.
- [ ] 데이터가 없을 경우 "저장소가 없습니다"라는 안내 문구가 표시된다.
