# 📋 구현 지침서 (Implementation Instructions)

이 디렉토리는 각 작업(Task)의 실제 구현을 위한 구체적인 가이드를 담고 있습니다.

## 💡 역할

- **Phase 3 산출물**: 요구사항(Requirement)을 시스템적으로 구현하기 위한 상세 기술 사양을 정의합니다.
- **개발 가이드**: 파일 구조, 함수 시그니처, 기술 제약 사항 등 실제 코딩 시 준수해야 할 세부 규칙을 포함합니다.
- **성공 기준**: 해당 작업이 완료되었음을 판단하는 기술적 근거와 테스트 시나리오를 명시합니다.

## 🛠️ 작업 프로세스

1. **지침서 확인**: `docs/requirements/tasks/` 문서를 통해 무엇을 만들지 확인하고, 본 폴더의 해당 TASK 파일을 읽어 기술 사양을 파악합니다.
2. **테스트 작성 (AI)**: 지침서의 상세 사양을 바탕으로 AI가 먼저 테스트 코드를 작성합니다.
3. **코드 구현 (사용자)**: 사용자가 실제 비즈니스 로직을 구현하여 테스트를 통과시킵니다.
4. **검증 및 완료**: 완료 판정 체크리스트를 확인하고 작업을 마무리합니다.

## 🔗 링크 규칙

- 각 작업 문서는 상단의 **연관 설계** 섹션을 통해 아키텍처 문서와 연결됩니다.
- 요구사항 문서의 '상세 지침' 링크를 통해 이 폴더의 파일로 이동할 수 있습니다.
