## 0. 현재 코드 상태와 이 작업의 위치

- **현재 상태 요약**: 스타일링 환경은 잡혔으나 상태 관리, 데이터 페칭 등 앱 구동에 필요한 핵심 라이브러리가 없습니다.
- **이 작업의 책임**: 프로젝트 전반에서 사용할 필수 라이브러리들을 설치하고, 모노레포 내부 패키지(`@chess-db/shared`)를 연결합니다.

- **이번 작업에서 하지 않는 것**: `[TASK-092]` (테스트 환경)에 연결된 후속 책임은 이번 태스크에서 함께 닫지 않는다.

- **경계 메모**:
  - 이번 태스크는 apps/web 필수 공통 라이브러리 의존성 추가 범위만 닫고, 후속 태스크 또는 인접 Feature의 세부 구현은 여기서 함께 처리하지 않는다.

## 🎯 1. 작업 목표

- **최종 상태**: `Zustand`, `TanStack Query` 등을 사용할 준비가 완료되며, `shared` 패키지의 도메인 로직을 웹 앱에서 가져다 쓸 수 있습니다.

- **이번 작업의 최소 결과물**:
  - `apps/web/package.json`

- **성공 기준 (AC)**:
  - 모든 필수 의존성이 `apps/web/package.json`에 명시되었다.
  - `shared` 패키지의 타입 추론이 IDE에서 정상적으로 이루어진다.
  - React Query Devtools가 개발 환경에서 정상 노출된다 (선택 사항).

## 🛠️ 3. 상세 기술 사양

- **설치 목록**:
  - 상태 관리: `zustand`
  - 서버 상태: `@tanstack/react-query`, `@tanstack/react-query-devtools`
  - 유틸리티: `clsx`, `tailwind-merge`, `lucide-react`
- **로컬 패키지 연결**: `pnpm add @chess-db/shared --filter web`
- **설정**: `QueryClientProvider`를 `App.tsx` 최상단에 배치.
- **필수 describe/it 목록**:
  - describe: `Library Integration`
    - it: `@chess-db/shared 패키지의 코드를 정상적으로 import 할 수 있어야 한다`
    - it: `Zustand 스토어가 React 컴포넌트와 유기적으로 연동되어야 한다`

- **핵심 조립/흐름 규칙**:
  - 모든 필수 의존성이 `apps/web/package.json`에 명시되었다.
  - `shared` 패키지의 타입 추론이 IDE에서 정상적으로 이루어진다.
  - React Query Devtools가 개발 환경에서 정상 노출된다 (선택 사항).

- **데이터 모델 해석**:
  - 핵심 데이터는 web 앱에서 공통으로 사용할 상태/라우팅/query/test 보조 라이브러리 목록과 버전입니다.
  - 입력은 앱 요구사항과 기술 스택 문서에서 정한 공통 라이브러리 범위입니다.

- **외부 의존성**:
  - `zustand`
  - `react-router-dom`
  - `@tanstack/react-query`
  - `zod`

- **import/export 규칙**:
  - `../architecture/project-rules.md`의 named export, 상대 경로 최소화, `.js` 확장자 규칙을 따른다.

- **권장 네이밍**:
  - 공개 함수/타입 이름: apps/web 필수 공통 라이브러리 의존성 추가 책임이 드러나는 이름
  - 내부 helper 이름: 역할이 바로 드러나는 동사형 또는 조합형 이름
  - 핵심 변수명: 상태와 대상이 분명한 이름
  - 피해야 할 이름: data, item, obj, temp

- **이름별 사용 의도와 적용 시점**:
  - `dependencies`와 `devDependencies` 구분은 실제 런타임 사용 여부를 기준으로 유지합니다.
  - 공통 라이브러리 목록은 후속 feature 문서가 가정하는 기반 계약입니다.

- **인수 이름 가이드**:
  - `packageName`, `versionRange`

- **짧은 예시 골격**:

```ts
pnpm --filter @chess-db/web add zustand react-router-dom @tanstack/react-query zod
```

- **최소 테스트 개수**:
  - 최소 3개

- **반드시 포함할 실패 시나리오**:
  - 후속 태스크가 가정한 라이브러리를 누락하는 경우
  - 런타임 의존성을 devDependencies에 넣는 경우

## ⚖️ 4. 기술 제약 및 규칙

- **작성 원칙**:
  - 전역 규칙을 반복 나열하지 말고, 이번 태스크에서 특히 강조해야 하는 제약만 짧게 적는다.

- **구조 규칙**:
  - 현재 디렉토리 구조와 연관 설계 문서의 책임 경계를 유지한다.

- **범위 규칙**:
  - `[TASK-092]` (테스트 환경)에 연결된 범위는 여기서 닫지 않는다.

- **헌법 정렬 규칙**:
  - `../architecture/project-rules.md`의 네이밍, import/export, 상태 규칙을 그대로 따른다.

- **문서화 규칙**:
  - 이 문서에서 고정한 파일 경로, 검증 기준, 후속 태스크 경계를 구현 단계와 동일하게 유지한다.
