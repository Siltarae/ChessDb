## 0. 현재 코드 상태와 이 작업의 위치

- **현재 상태 요약**: 폰과 나이트의 이동 로직은 구현되었으나, 장애물에 막히는 '슬라이딩(Sliding)' 기물 로직은 부재합니다.
- **이 작업의 책임**: 비숍의 대각선 이동 로직을 구현합니다. 특히 다른 기물에 의해 경로가 차단되는 '슬라이딩' 규칙을 정확히 처리해야 합니다.

## 🎯 1. 작업 목표

- **최종 상태**: 비숍이 4개 대각선 방향으로 장애물 전까지 이동하거나 상대 기물을 캡처할 수 있는 모든 위치를 반환합니다.

## 🛠️ 3. 상세 기술 사양

- **공개 함수 시그니처**:
  ```ts
  export function getBishopMoves(square: Square, state: GameState): Square[];
  ```
- **핵심 조립/흐름 규칙**:
  - 4개 대각선 방향 오프셋: `[1, 1]`, `[1, -1]`, `[-1, 1]`, `[-1, -1]`
  - 각 방향으로 한 칸씩 확장하며 검사:
    - 빈 칸: 이동 가능 목록에 추가하고 계속 확장.
    - 상대 기물: 이동 가능 목록에 추가(캡처)하고 해당 방향 확장 중단.
    - 아군 기물: 해당 방향 확장 즉시 중단.
    - 보드 경계: 확장 중단.

## ✅ 4. 완료 판정 체크리스트

- [ ] 4방향 대각선 슬라이딩 이동이 정확히 계산된다.
- [ ] 자신의 기물에 막혔을 때 해당 방향의 더 먼 칸이 계산에서 제외된다.
- [ ] 상대 기물을 잡는 칸까지는 이동 가능 목록에 포함된다.
