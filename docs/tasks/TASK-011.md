## 0. 현재 코드 상태와 이 작업의 위치

- **현재 상태 요약**: 각 기물별 '의사 합법 수(Pseudo-legal moves)'와 체크 상태 판정(`isCheck`) 로직이 준비되었습니다. 하지만 킹을 스스로 위험에 빠뜨리는 수를 걸러내는 로직은 부재합니다.
- **이 작업의 책임**: 의사 합법 수 중 '자신의 킹을 체크 상태로 만드는 수'를 제외하여 최종적인 **합법 수(Legal Moves)** 목록을 생성합니다.
- **경계 메모**: 핀(Pin)에 걸린 기물의 이동 제한과 체크 상황에서의 탈출 경로 확보를 모두 이 단계에서 처리합니다.

- **이번 작업에서 하지 않는 것**: `[TASK-012]` (캐슬링)에 연결된 후속 책임은 이번 태스크에서 함께 닫지 않는다.

- **이번 작업의 최소 결과물**:
  - `packages/shared/src/engines/legal-move-engine.ts`
  - `packages/shared/src/engines/legal-move-engine.spec.ts`
- **성공 기준 (AC)**:
  - 킹의 안전이 100% 보장되는 수만 반환됨을 테스트로 확인했다.
  - 핀 상태의 기물 이동 제한이 정확히 적용되었다.
  - 가상 착수 로직에서 원본 상태(State)가 오염되지 않음을 확인했다.

## 🛠️ 3. 상세 기술 사양

- **핵심 조립/흐름 규칙**:
  1. 특정 위치(`square`) 기물의 모든 '의사 합법 수'를 가져온다.
  2. 각 수(to)에 대해:
     - '가상 착수' 후의 임시 보드 상태를 생성한다.
     - `isCheck(nextState, currentTurnColor)`를 호출한다.
     - 자신의 킹이 여전히(또는 새롭게) 체크 상태라면 해당 수를 목록에서 제외한다.
- **공개 함수 시그니처**:
  ```ts
  import { GameState, Square } from '../models/game-state.js';
  export function getLegalMoves(square: Square, state: GameState): Square[];
  ```
- **필수 describe/it 목록**:
  - describe: `getLegalMoves`
    - it: `핀(Pin)에 걸린 기물이 킹을 노출시키며 이동하는 것을 차단해야 한다`
    - it: `체크 상태일 때 체크를 해소하는 수만 반환해야 한다`
    - it: `킹 스스로가 상대 공격 경로로 이동하는 것을 차단해야 한다`

- **데이터 모델 해석**:
  - 입력은 후보 수 집합과 현재 `GameState`이며, 출력은 체크를 유발하지 않는 합법 수 집합입니다.
  - 자기 킹이 체크 상태에 남는 수는 모두 걸러야 합니다.

- **외부 의존성**:
  - shared 내부 `executeMove`, `isCheck` 또는 동등 엔진
  - `vitest`

- **import/export 규칙**:
  - `../architecture/project-rules.md`의 named export, 상대 경로 최소화, `.js` 확장자 규칙을 따른다.

- **권장 네이밍**:
  - 공개 함수/타입 이름: 체크 상태 기준 합법 수 필터링 책임이 드러나는 이름
  - 내부 helper 이름: 역할이 바로 드러나는 동사형 또는 조합형 이름
  - 핵심 변수명: 상태와 대상이 분명한 이름
  - 피해야 할 이름: data, item, obj, temp

- **이름별 사용 의도와 적용 시점**:
  - `getLegalMoves`는 특정 기물의 후보 수를 최종 합법 수로 좁힐 때 사용합니다.
  - `simulateMove`는 내부 검증용 임시 상태 계산 helper 이름으로 사용할 수 있습니다.

- **인수 이름 가이드**:
  - `state`, `fromSquare`, `candidateMoves`

- **짧은 예시 골격**:

```ts
const legalMoves = getLegalMoves(state, square);
```

- **최소 테스트 개수**:
  - 최소 3개

- **반드시 포함할 실패 시나리오**:
  - 후보 수를 그대로 반환해 자충수를 허용하는 경우
  - 시뮬레이션 과정에서 원본 상태를 오염시키는 경우
