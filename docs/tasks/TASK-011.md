## 0. 현재 코드 상태와 이 작업의 위치

- **현재 상태 요약**: 각 기물별 '의사 합법 수(Pseudo-legal moves)'와 체크 상태 판정(`isCheck`) 로직이 준비되었습니다. 하지만 킹을 스스로 위험에 빠뜨리는 수를 걸러내는 로직은 부재합니다.
- **이 작업의 책임**: 의사 합법 수 중 '자신의 킹을 체크 상태로 만드는 수'를 제외하여 최종적인 **합법 수(Legal Moves)** 목록을 생성합니다.
- **경계 메모**: 핀(Pin)에 걸린 기물의 이동 제한과 체크 상황에서의 탈출 경로 확보를 모두 이 단계에서 처리합니다.

## 🎯 1. 작업 목표

- **최종 상태**: `getLegalMoves(square, state)` 함수가 체스 규칙상 완벽히 허용되는 수만 반환하여, 사용자 UI에서 잘못된 수를 두지 못하도록 방어합니다.

## 🛠️ 3. 상세 기술 사양

- **핵심 조립/흐름 규칙**:
  1. 특정 위치(`square`) 기물의 모든 '의사 합법 수'를 가져온다.
  2. 각 수(to)에 대해:
     - '가상 착수' 후의 임시 보드 상태를 생성한다.
     - `isCheck(nextState, currentTurnColor)`를 호출한다.
     - 자신의 킹이 여전히(또는 새롭게) 체크 상태라면 해당 수를 목록에서 제외한다.
- **공개 함수 시그니처**:
  ```ts
  import { GameState, Square } from '../models/game-state.js';
  export function getLegalMoves(square: Square, state: GameState): Square[];
  ```
- **필수 describe/it 목록**:
  - describe: `getLegalMoves`
    - it: `핀(Pin)에 걸린 기물이 킹을 노출시키며 이동하는 것을 차단해야 한다`
    - it: `체크 상태일 때 체크를 해소하는 수만 반환해야 한다`
    - it: `킹 스스로가 상대 공격 경로로 이동하는 것을 차단해야 한다`
