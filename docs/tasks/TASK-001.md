## 0. 현재 코드 상태와 이 작업의 위치

- **현재 상태 요약**: 기초 프레임워크와 스타일링 도구는 준비되었으나, 실제 체스 기보 입력을 위한 레이아웃은 없는 상태입니다.
- **이 작업의 책임**: 체스 보드, 수순 목록, 코멘트 영역이 배치될 전체적인 페이지 레이아웃을 구축합니다.

## 🎯 1. 작업 목표

- **최종 상태**: 사용자가 체스 기보 입력을 위해 필요한 모든 UI 영역(보드, 수순, 도구)이 적절한 위치에 배치된 화면을 봅니다.
- **핵심 산출물**: `NotationInputPage.tsx`, `ChessBoardContainer.tsx`, `MoveListContainer.tsx`

## 🛠️ 3. 상세 기술 사양

- **레이아웃 구조**:
  - 데스크탑: 좌측(체스 보드), 우측 상단(수순 목록), 우측 하단(코멘트 및 도구).
  - 모바일: 상단(체스 보드), 하단(탭 형식으로 수순/코멘트 전환).
- **권장 네이밍**: `NotationInputView`, `BoardSection`, `SidePanel`

## ✅ 4. 완료 판정 체크리스트

- [ ] 각 영역이 `shadcn/ui`의 Card 또는 Layout 컴포넌트로 구분되어 배치되었다.
- [ ] 반응형 레이아웃이 적용되어 모바일/데스크탑 모두 가독성이 확보되었다.
