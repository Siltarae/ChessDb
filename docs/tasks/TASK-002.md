## 0. 현재 코드 상태와 이 작업의 위치

- **현재 상태 요약**: 화면 레이아웃은 준비되었으나 보드 내부는 빈 영역입니다. `shared` 패키지의 데이터 모델은 이미 준비되어 있습니다.
- **이 작업의 책임**: 8x8 체스판 격자를 그리고, 초기 기물 배치를 화면에 렌더링합니다.

## 🎯 1. 작업 목표

- **최종 상태**: 사용자가 브라우저에서 실제 체스판과 표준 시작 위치에 놓인 기물들을 시각적으로 확인합니다.

## 🛠️ 3. 상세 기술 사양

- **좌표 매핑**:
  - `shared`의 인덱스 0(a1)은 화면의 좌측 하단이어야 함.
  - 행(Rank): 0~7 (1~8), 열(File): 0~7 (a~h).
- **스타일링**:
  - 밝은 칸과 어두운 칸이 교차하도록 `nth-child` 또는 인덱스 짝홀수 로직 적용.
  - 기물 이미지는 오픈 소스 SVG (예: Wikipedia Chess Pieces) 또는 `lucide-react` 아이콘 활용.
- **상태 연동**:
  - `useChessStore`에서 현재 `board` 배열을 구독하여 렌더링.
- **필수 describe/it 목록**:
  - describe: `Board Rendering`
    - it: `정확히 64개의 Square 컴포넌트가 생성되어야 한다`
    - it: `초기 상태에서 32개의 기물이 정확한 위치에 표시되어야 한다`
    - it: `좌표 라벨(a-h, 1-8)이 보드 경계에 나타나야 한다`
