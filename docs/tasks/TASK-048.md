## 0. 현재 코드 상태와 이 작업의 위치

- **현재 상태 요약**: 기보 입력 뷰(`FEATURE-002`)와 기보 상세 뷰(`FEATURE-005`)는 존재하나, 통계 조회를 위한 독립적인 '분석 보드' 전용 뷰는 없는 상태입니다.
- **이 작업의 책임**: 사용자가 저장소에 진입했을 때 처음 보게 될 분석 보드 화면을 구성하고, 보드에서 직접 수를 두어 특정 포지션을 만드는 기능을 구현합니다.

## 🎯 1. 작업 목표

- **최종 상태**: 사용자가 분석 보드에서 자유롭게 수를 두며 탐색하고, 현재 보드 상태(FEN)가 통계 조회 시스템과 연동될 준비가 됩니다.
- **핵심 산출물**: `AnalysisPageView.tsx`, `AnalysisBoardContainer.tsx`

## 🛠️ 3. 상세 기술 사양

- **상태 관리**: `useAnalysisStore` (Zustand) 생성. `gameState`와 `history`를 포함하되, 정식 저장과는 독립된 '탐색용' 상태로 유지.
- **입력 로직**: 기보 입력(`TASK-016`)과 동일한 `makeMove` 엔진을 사용하지만, 이력을 서버에 저장하지 않고 로컬 상태에서만 유지.
- **UI**: 보드 우측에 '통계 패널' 영역을 확보하여 추후 `TASK-058`에서 결과가 표시될 수 있게 레이아웃 구성.

## ✅ 4. 완료 판정 체크리스트

- [ ] 저장소 진입 시 분석 보드 화면이 가장 먼저 노출된다.
- [ ] 보드 위에서 기물을 자유롭게 움직여 원하는 포지션을 만들 수 있다.
- [ ] 분석 보드에서의 움직임은 정식 기보 저장으로 이어지지 않고 오직 '탐색'용으로만 작동한다.
