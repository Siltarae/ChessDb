## 0. 현재 코드 상태와 이 작업의 위치

- **현재 상태 요약**: 기보 입력 뷰와 상세 뷰는 완성되었습니다. 통계 데이터와 연동되어 자유롭게 수를 두어볼 수 있는 독립적인 '분석 전용' 화면이 필요합니다.
- **이 작업의 책임**: 사용자가 저장소에 진입했을 때 처음 보게 될 분석 보드 화면을 구성하고, 보드에서 직접 기물을 움직여 특정 포지션을 만드는 인터랙션을 구현합니다.

## 🎯 1. 작업 목표

- **최종 상태**: 사용자가 분석 보드에서 자유롭게 탐색하며 수를 두면, 우측의 통계 패널(추후 구현)에 해당 포지션의 정보가 실시간으로 동기화될 준비가 됩니다.

## 🛠️ 3. 상세 기술 사양

- **상태 관리**:
  - `useAnalysisStore` (Zustand) 생성.
  - `gameState`, `currentFen` 상태 관리.
  - 정식 기보 저장과는 완전히 분리된 독립된 샌드박스 상태입니다.
- **입력 로직**:
  - 기보 입력(`TASK-016`)에서 사용한 엔진 로직을 동일하게 사용.
  - 단, 분석 보드에서는 모든 기물의 색상을 자유롭게 움직일 수 있거나(Setup 모드) 턴 제한을 유연하게 가져갈 수 있도록 옵션 고려.
- **UI 레이아웃**:
  - 좌측: 체스 보드.
  - 우측: 통계 정보 및 수 후보 목록 패널 (레이아웃만 먼저 구성).

## ✅ 7. 완료 판정 체크리스트

- [ ] 저장소 진입 시 가장 먼저 분석 보드 화면이 노출된다.
- [ ] 보드 위에서 기물을 움직였을 때 현재 포지션(FEN)이 정확히 계산되어 상태에 반영된다.
- [ ] 분석 보드에서의 움직임은 정식 기보 데이터에 영향을 주지 않는다.
