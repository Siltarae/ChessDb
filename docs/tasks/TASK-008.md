## 0. 현재 코드 상태와 이 작업의 위치

- **현재 상태 요약**: 비숍과 룩의 이동 로직이 완성되었으며, `getSlidingMoves` 공통 엔진이 안정적으로 작동하고 있습니다.
- **이 작업의 책임**: 비숍과 룩의 모든 방향(8방향)을 합쳐 체스에서 가장 강력한 기물인 퀸(Queen)의 이동 로직을 구현합니다.

- **이번 작업에서 하지 않는 것**: `[TASK-009]` (킹 로직)에 연결된 후속 책임은 이번 태스크에서 함께 닫지 않는다.

- **경계 메모**:
  - 이번 태스크는 퀸(Queen) 합법 수 판정 범위만 닫고, 후속 태스크 또는 인접 Feature의 세부 구현은 여기서 함께 처리하지 않는다.

## 🎯 1. 작업 목표

- **최종 상태**: 퀸이 8개 전 방향으로 장애물 전까지 이동하거나 상대 기물을 캡처할 수 있는 모든 좌표 배열을 반환합니다.

- **이번 작업의 최소 결과물**:
  - `packages/shared/src/engines/queen-engine.ts`
  - `packages/shared/src/engines/queen-engine.spec.ts`
  - `packages/shared/src/engines/sliding-engine.ts`
- **성공 기준 (AC)**:
  - `getQueenMoves`가 공통 슬라이딩 엔진을 통해 정확히 구현되었다.
  - 8개 방향 모두에 대해 누락 없이 이동 가능 칸이 반환된다.
  - 모든 import에 `.js` 확장자가 포함되어 있다.

## 🛠️ 3. 상세 기술 사양

- **방향 벡터**: `BISHOP_DIRECTIONS`와 `ROOK_DIRECTIONS`를 합친 8개의 `{dx, dy}` 쌍.
- **구현 방식**: 별도의 독자적인 로직 없이, 이미 검증된 `getSlidingMoves` 엔진에 8방향 벡터를 전달하여 결과를 도출합니다.
- **필수 describe/it 목록**:
  - describe: `getQueenMoves`
    - it: `빈 보드 중앙(D4)에서 27개의 이동 가능 위치를 반환해야 한다 (직선 14 + 대각선 13)`
    - it: `복합적인 장애물 상황에서 정확한 이동 범위를 계산해야 한다`
- **최소 테스트 개수**: 2개 이상

- **핵심 조립/흐름 규칙**:
  - `getQueenMoves`가 공통 슬라이딩 엔진을 통해 정확히 구현되었다.
  - 8개 방향 모두에 대해 누락 없이 이동 가능 칸이 반환된다.
  - 모든 import에 `.js` 확장자가 포함되어 있다.

- **데이터 모델 해석**:
  - 퀸 이동은 비숍 대각선 규칙과 룩 직선 규칙의 합집합입니다.
  - 반환값은 8방향 전체를 포함한 `Square[]`입니다.

- **외부 의존성**:
  - shared 내부 `sliding-engine`, `rook-engine`, `bishop-engine` 참고 구조
  - `vitest`

- **import/export 규칙**:
  - `../architecture/project-rules.md`의 named export, 상대 경로 최소화, `.js` 확장자 규칙을 따른다.

- **권장 네이밍**:
  - 공개 함수/타입 이름: 퀸(Queen) 합법 수 판정 책임이 드러나는 이름
  - 내부 helper 이름: 역할이 바로 드러나는 동사형 또는 조합형 이름
  - 핵심 변수명: 상태와 대상이 분명한 이름
  - 피해야 할 이름: data, item, obj, temp

- **이름별 사용 의도와 적용 시점**:
  - `queenDirections`는 8방향 상수 이름으로 사용합니다.
  - `getQueenMoves`는 퀸 전용 공개 API 이름으로 유지합니다.

- **인수 이름 가이드**:
  - `square`, `state`

- **짧은 예시 골격**:

```ts
const moves = getQueenMoves(square, state);
```

- **반드시 포함할 실패 시나리오**:
  - 직선 또는 대각선 한쪽만 계산하는 경우
  - 방향 집합에 중복이 있어 같은 칸이 두 번 반환되는 경우
