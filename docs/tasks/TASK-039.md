## 0. 현재 코드 상태와 이 작업의 위치

- **현재 상태 요약**: 기보 입력용 보드 UI는 존재하지만, 이미 저장된 '완료된 기보'를 로드하여 읽기 전용으로 보여주는 전용 화면은 없습니다.
- **이 작업의 책임**: 저장된 기보 데이터를 불러와 보드와 수순 목록에 뿌려주고, 사용자가 수순을 클릭하면 해당 시점의 보드로 즉시 이동하는 '복기' 기능을 구현합니다.

## 🎯 1. 작업 목표

- **최종 상태**: 사용자가 목록에서 기보를 선택하면 전체 수순이 로드되며, 수순 목록이나 키보드 방향키를 통해 한 수씩 넘겨가며 대국을 검토할 수 있습니다.

## 🛠️ 3. 상세 기술 사양

- **읽기 전용 모드**:
  - `Board` 컴포넌트에 `readOnly` prop을 전달하여 기물 드래그 앤 드롭 기능을 비활성화합니다.
- **수순 탐색 로직**:
  - `currentMoveIndex` 상태를 관리 (초기값: 0).
  - 특정 수 클릭 시 `currentMoveIndex` 업데이트.
  - 초기 보드 상태부터 `currentMoveIndex`까지의 수순을 `executeMove`로 순차 실행하여 현재 화면에 그릴 `GameState` 도출.
- **UI/UX**:
  - `MoveList` 컴포넌트 재사용, 현재 보고 있는 수 하이라이트.
  - 좌/우 방향키 입력 시 `currentMoveIndex` 증감 처리.

## ✅ 7. 완료 판정 체크리스트

- [ ] 저장된 기보의 모든 수순이 목록에 번호와 함께 정확히 표시된다.
- [ ] 수순 클릭 또는 방향키 조작 시 보드 배치가 해당 시점으로 즉시 동기화된다.
- [ ] 읽기 전용 모드에서 새로운 수를 두려는 시도가 불가능함을 확인했다.
