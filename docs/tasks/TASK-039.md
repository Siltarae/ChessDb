## 0. 현재 코드 상태와 이 작업의 위치

- **현재 상태 요약**: 기보 입력 보드(`FEATURE-002`)는 존재하지만, 이미 저장된 '완료된 기보'를 읽기 전용 모드로 보여주는 상세 화면은 없습니다.
- **이 작업의 책임**: 저장된 기보 데이터를 불러와 보드와 수순 목록에 뿌려주고, 사용자가 수순을 클릭하면 해당 시점의 보드로 이동하는 상호작용을 구현합니다.

## 🎯 1. 작업 목표

- **최종 상태**: 사용자가 목록에서 기보를 선택하면 해당 기보의 전체 수순이 로드되며, 수순을 하나씩 넘기며 복기할 수 있습니다.

## 🛠️ 3. 상세 기술 사양

- **읽기 전용 모드**: 상세 화면에서는 기물을 드래그하여 새로운 수를 두는 기능을 비활성화합니다.
- **수순 탐색 로직**:
  - 사용자가 수순 목록의 `N`번째 수를 클릭.
  - 초기 상태부터 `N`번째 수까지 `makeMove`를 반복 실행하여 해당 시점의 보드 상태 계산 및 렌더링.
- **UI**: `shared`의 수순 데이터를 기반으로 `MoveList` 컴포넌트 재사용.

## ✅ 4. 완료 판정 체크리스트

- [ ] 저장된 기보의 모든 수순이 목록에 정확히 표시된다.
- [ ] 수순 목록의 수를 클릭하면 보드가 해당 시점의 배치로 즉시 업데이트된다.
- [ ] 키보드 방향키(좌/우)를 통한 수순 탐색 기능이 작동한다 (선택 사항).
