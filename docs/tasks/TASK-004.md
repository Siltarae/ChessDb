## 0. 현재 코드 상태와 이 작업의 위치

- **현재 상태 요약**: 데이터 모델만 정의되어 있으며 이동 로직은 전무합니다.
- **이 작업의 책임**: 폰의 전진, 첫 수 2칸 전진, 대각선 캡처 로직을 구현합니다. (앙파상과 프로모션은 별도 태스크에서 처리)

## 🎯 1. 작업 목표

- **최종 상태**: 특정 위치의 폰이 현재 보드 상태에서 이동할 수 있는 기초적인 합법 수(Pseudo-legal moves)를 반환합니다.

## 🛠️ 3. 상세 기술 사양

- **공개 함수 시그니처**:
  ```ts
  export function getPawnMoves(square: Square, state: GameState): Square[];
  ```
- **핵심 규칙**:
  - 흰색 폰은 행 증가, 검은색 폰은 행 감소 방향으로 이동.
  - 전진 시 기물이 있으면 이동 불가.
  - 대각선 1칸에 상대 기물이 있을 때만 캡처 이동 가능.
- **필수 테스트 목록**:
  - [ ] 전진 1칸 (장애물 없을 시)
  - [ ] 첫 수 2칸 전진 (장애물 없을 시)
  - [ ] 대각선 캡처 (상대 기물 존재 시)
  - [ ] 전진 차단 (자신의 기물 또는 상대 기물 존재 시)

## ✅ 4. 완료 판정 체크리스트

- [ ] 폰의 기본 이동 및 캡처 규칙이 테스트를 통과한다.
- [ ] 색상별(White/Black) 이동 방향이 정확히 처리되었다.
