## 0. 현재 코드 상태와 이 작업의 위치

- **현재 상태 요약**: 수순 이력은 쌓이지만, 과거 시점으로 돌아가거나 실수를 취소하는 기능은 없습니다.
- **이 작업의 책임**: 수순 목록에서 특정 시점을 선택하거나 전용 버튼을 눌러 보드 상태를 과거/미래로 이동시킵니다.

## 🎯 1. 작업 목표

- **최종 상태**: 사용자가 'Undo'를 누르면 한 수 전으로, 'Redo'를 누르면 다시 앞수로 이동하며, 수순 목록의 특정 수를 클릭해도 해당 시점이 즉시 렌더링됩니다.

## 🛠️ 3. 상세 기술 사양

- **탐색 로직**:
  - `history` 배열은 보존한 채 `currentPointer`를 이동시켜 현재 화면에 보여줄 시점을 결정합니다.
  - **분기 처리**: 과거 시점에서 새로운 수를 두면, 포인터 이후의 모든 수순을 삭제(`truncate`)하고 새로운 경로를 시작합니다.
- **UI**:
  - 보드 하단에 컨트롤 바(<<, <, >, >>) 컴포넌트 배치.
  - 키보드 방향키 이벤트 리스너 연동.

## ✅ 7. 완료 판정 체크리스트

- [ ] 되돌리기(Undo) 시 보드 상태와 턴이 이전 수순으로 완벽히 복구된다.
- [ ] 수순 목록의 특정 수를 클릭하면 보드가 해당 배치로 즉시 업데이트된다.
- [ ] 과거 시점에서 새 수 착수 시 '미래 수순 삭제' 로직이 정확히 작동한다.
