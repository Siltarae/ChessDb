## 🛠️ 3. 상세 기술 사양

- **방향 벡터**: `[{dx:0, dy:1}, {dx:0, dy:-1}, {dx:1, dy:0}, {dx:-1, dy:0}]`
- **구현 방식**: `getSlidingMoves(square, ROOK_DIRECTIONS, state)`와 같이 공통 엔진을 호출하여 로직 중복을 최소화합니다.
- **필수 describe/it 목록**:
  - describe: `getRookMoves`
    - it: `빈 보드 중앙(D4)에서 14개의 수평/수직 위치를 반환해야 한다`
    - it: `동일한 파일 상의 기물에 의해 이동이 제한되어야 한다`
    - it: `상대 기물을 캡처하며 멈추는지 확인해야 한다`

- **핵심 조립/흐름 규칙**:
  - 룩의 4방향 이동이 정확히 반환된다.
  - `getSlidingMoves` 엔진을 재사용하여 DRY 원칙을 지켰다.
  - 모든 테스트가 통과한다.

- **데이터 모델 해석**:
  - `GameState.board`와 직선 4방향 벡터를 사용합니다.
  - 반환값은 룩의 직선 이동 가능 칸 배열입니다.

- **외부 의존성**:
  - shared 내부 `sliding-engine` 재사용
  - `vitest`

- **import/export 규칙**:
  - `../architecture/project-rules.md`의 named export, 상대 경로 최소화, `.js` 확장자 규칙을 따른다.

- **권장 네이밍**:
  - 공개 함수/타입 이름: 룩(Rook) 합법 수 판정 책임이 드러나는 이름
  - 내부 helper 이름: 역할이 바로 드러나는 동사형 또는 조합형 이름
  - 핵심 변수명: 상태와 대상이 분명한 이름
  - 피해야 할 이름: data, item, obj, temp

- **이름별 사용 의도와 적용 시점**:
  - `rookDirections`는 수평/수직 4방향 상수 이름으로 사용합니다.
  - `getRookMoves`는 룩 전용 공개 함수 이름으로 유지합니다.

- **인수 이름 가이드**:
  - `square`, `state`, `directions`

- **짧은 예시 골격**:

```ts
const moves = getRookMoves(square, state);
```

- **최소 테스트 개수**:
  - 최소 3개

- **반드시 포함할 실패 시나리오**:
  - 대각선 칸이 결과에 섞이는 경우
  - 중간 장애물을 지나친 칸이 반환되는 경우
