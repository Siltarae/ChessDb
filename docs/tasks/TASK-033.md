## 0. 현재 코드 상태와 이 작업의 위치

- **현재 상태 요약**: 목록 페이지와 개별 입력 페이지는 존재하지만, 특정 저장소의 맥락(ID)을 가지고 페이지 간을 이동하는 경로 시스템이 연결되지 않았습니다.
- **이 작업의 책임**: 특정 저장소 카드를 클릭했을 때 해당 저장소 전용 화면(분석 보드)으로 이동하고, 거기서 다시 기보 입력으로 전환하는 라우팅 흐름을 구축합니다.

## 🎯 1. 작업 목표

- **최종 상태**: 사용자가 저장소를 선택해 들어가면 해당 저장소의 기보들을 관리할 수 있는 메인 대시보드(분석 뷰)를 보게 됩니다.

## 🛠️ 3. 상세 기술 사양

- **라우팅 구성 (React Router v6)**:
  - `/repositories`: 저장소 전체 목록.
  - `/repositories/:repoId`: 특정 저장소의 분석 보드 및 기보 목록.
  - `/repositories/:repoId/new`: 해당 저장소에 새 기보 추가.
- **화면 전환 로직**:
  - `useNavigate`를 사용하여 카드 클릭 시 상세 경로로 이동.
  - 상세 화면 상단에 '새 기보 작성' 버튼을 배치하여 입력 페이지로 유도.
- **상태 연동**: `repoId` 파라미터를 사용하여 해당 저장소에 속한 기보들만 필터링하여 조회.

## ✅ 7. 완료 판정 체크리스트

- [ ] 저장소 카드 클릭 시 URL에 `repoId`가 포함된 상세 경로로 정확히 이동한다.
- [ ] 저장소 상세 페이지 진입 시 해당 저장소의 이름이 헤더 등에 올바르게 표시된다.
- [ ] '새 기보 작성' 클릭 시 기존에 구축한 입력 화면으로 저장소 맥락을 유지하며 전환된다.
