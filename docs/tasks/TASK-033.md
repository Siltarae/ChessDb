## 0. 현재 코드 상태와 이 작업의 위치

- **현재 상태 요약**: 각 페이지(목록, 입력)는 존재하지만, 특정 저장소의 맥락을 가지고 페이지 간을 이동하는 라우팅 시스템이 연결되지 않았습니다.
- **이 작업의 책임**: 특정 저장소를 클릭했을 때 해당 저장소의 ID를 경로 파라미터로 가지고 분석 보드 뷰로 진입하며, 거기서 새 기보 입력을 시작하는 흐름을 연결합니다.

## 🎯 1. 작업 목표

- **최종 상태**: 사용자가 저장소를 선택해 들어가면 해당 저장소 전용 화면(분석 보드)을 보게 되며, 버튼 하나로 기보 입력 모드로 전환할 수 있습니다.

## 🛠️ 3. 상세 기술 사양

- **라우팅 (React Router)**:
  - `/repositories` (목록)
  - `/repositories/:repoId` (저장소 메인 - 분석 보드)
  - `/repositories/:repoId/new` (기보 입력)
- **화면 고정**: 저장소 진입 시 첫 화면은 무조건 분석 보드(`AnalysisView`)여야 함.
- **전환 버튼**: 분석 보드 화면에 `새 기보 작성` 버튼을 배치하여 입력 URL로 이동.

## ✅ 4. 완료 판정 체크리스트

- [ ] 저장소 카드 클릭 시 해당 저장소의 상세 URL로 이동한다.
- [ ] 진입 시 분석 보드 영역이 가장 먼저 노출된다.
- [ ] '새 기보 작성' 클릭 시 기존에 구축한 기보 입력 화면(`FEATURE-002`)으로 자연스럽게 전환된다.
