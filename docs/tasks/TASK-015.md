**작업 상태**: 대기 중  
**선행 작업**: `[TASK-003]`, `[TASK-011]`  
**후속 작업**: `[TASK-016]`

---

## 0. 현재 코드 상태와 이 작업의 위치

- **현재 상태 요약**: 보드 렌더링과 기본 상태 관리는 가능하지만, 사용자의 클릭 상호작용이 정의되지 않았습니다.
- **이 작업의 책임**: 사용자가 기물을 클릭했을 때 해당 기물을 '선택' 상태로 만들고, `shared` 엔진을 호출하여 합법 수 위치에 하이라이트를 표시합니다.

## 🎯 1. 작업 목표

- **최종 상태**: 기물 클릭 시 선택된 칸은 강조되고, 이동 가능한 칸들에는 점(Dot)이나 반투명 오버레이가 표시됩니다.

## 🛠️ 3. 상세 기술 사양

- **상태 추가**: `selectedSquare: Square | null`
- **로직 흐름**:
  1. `Square` 클릭 이벤트 발생.
  2. 해당 칸에 현재 턴의 기물이 있으면 `selectedSquare`로 설정.
  3. `shared` 패키지의 `getLegalMoves(selectedSquare, gameState)` 호출.
  4. 반환된 좌표 배열을 보드 컴포넌트에 전달하여 렌더링.

## ✅ 4. 완료 판정 체크리스트

- [ ] 기물 클릭 시 선택 하이라이트가 정상적으로 표시된다.
- [ ] `shared` 엔진에서 계산된 합법 수 위치에만 하이라이트가 나타난다.
- [ ] 빈 칸이나 상대 기물 클릭 시(선택 전)에는 하이라이트가 발생하지 않는다.
