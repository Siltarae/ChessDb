## 0. 현재 코드 상태와 이 작업의 위치

- **현재 상태 요약**: 보드 표시와 기물 배치까지 가능하지만, 사용자의 클릭에 반응하는 상호작용은 없는 상태입니다.
- **이 작업의 책임**: 사용자가 기물을 클릭했을 때 해당 칸을 강조하고, 이동 가능한 합법 수 위치에 시각적 가이드를 표시합니다.

- **이번 작업에서 하지 않는 것**: `[TASK-016]` (착수 실행)에 연결된 후속 책임은 이번 태스크에서 함께 닫지 않는다.

- **경계 메모**:
  - 이번 태스크는 선택 기물 합법 수 하이라이트 범위만 닫고, 후속 태스크 또는 인접 Feature의 세부 구현은 여기서 함께 처리하지 않는다.
  - 선택 칸과 합법 수 칸은 `FEATURE-002` 스케치의 선택 칸 강조와 작은 점(dot) 표현을 기준으로 구분한다.

## 🎯 1. 작업 목표

- **최종 상태**: 기물을 클릭하면 선택된 칸이 하이라이트되고, 갈 수 있는 모든 칸에 작은 점(Dot)이나 반투명 효과가 나타납니다.

- **이번 작업의 최소 결과물**:
  - `apps/web/src/features/legal-move-highlight/model/use-legal-move-highlight.ts`
  - `apps/web/src/widgets/chess-board/ui/chess-board.tsx`
  - `apps/web/src/pages/notation-input-page.tsx`
- **성공 기준 (AC)**:
  - 아군 기물 클릭 시 해당 칸과 이동 가능 칸이 즉시 하이라이트된다.
  - 빈 칸이나 상대 기물(선택 전) 클릭 시에는 하이라이트가 발생하지 않는다.
  - 다른 기물을 클릭하면 이전 하이라이트가 사라지고 새로운 기물 기준으로 갱신된다.

## 🛠️ 3. 상세 기술 사양

- **상태 관리**:
  - `selectedSquare: Square | null` 상태를 스토어에 추가.
- **인터랙션 흐름**:
  1. 기물이 있는 칸 클릭 시 `selectedSquare` 업데이트.
  2. 이미 선택된 칸을 다시 클릭하면 선택 해제.
  3. 선택된 상태에서 `shared` 엔진의 `getLegalMoves(selectedSquare, gameState)` 호출.
  4. 반환된 좌표 배열을 활용하여 `Square` 컴포넌트에 `isHighlighted` prop 전달.
- **UI**:
  - 선택된 칸: 테두리 강조 또는 배경색 변경.
  - 합법 수 칸: 중앙에 작은 원(Circle) 오버레이 표시.

- **핵심 조립/흐름 규칙**:
  - 아군 기물 클릭 시 해당 칸과 이동 가능 칸이 즉시 하이라이트된다.
  - 빈 칸이나 상대 기물(선택 전) 클릭 시에는 하이라이트가 발생하지 않는다.
  - 다른 기물을 클릭하면 이전 하이라이트가 사라지고 새로운 기물 기준으로 갱신된다.

- **데이터 모델 해석**:
  - 입력은 현재 선택된 칸, 현재 보드 상태, 선택 기물의 합법 수 배열입니다.
  - 출력은 보드 위젯에 그릴 하이라이트 좌표와 선택 상태입니다.

- **외부 의존성**:
  - `react`
  - `zustand` 또는 feature-local state
  - `@chess-db/shared` 이동 엔진 결과

- **import/export 규칙**:
  - `../architecture/project-rules.md`의 named export, 상대 경로 최소화, `.js` 확장자 규칙을 따른다.

- **권장 네이밍**:
  - 공개 함수/타입 이름: 선택 기물 합법 수 하이라이트 책임이 드러나는 이름
  - 내부 helper 이름: 역할이 바로 드러나는 동사형 또는 조합형 이름
  - 핵심 변수명: 상태와 대상이 분명한 이름
  - 피해야 할 이름: data, item, obj, temp

- **이름별 사용 의도와 적용 시점**:
  - `useLegalMoveHighlight`는 선택 칸과 하이라이트 좌표를 함께 노출할 때 사용합니다.
  - `highlightSquares`는 화면에 칠할 칸 목록을 뜻할 때 사용합니다.

- **인수 이름 가이드**:
  - `selectedSquare`, `highlightSquares`, `boardState`

- **짧은 예시 골격**:

```ts
const { selectedSquare, highlightSquares } = useLegalMoveHighlight();
```

- **필수 describe/it 목록**:
  - 최상위 describe: `선택 기물 합법 수 하이라이트`
  - 필수 it: 요구사항 문서의 완료 기준을 직접 검증하는 테스트

- **최소 테스트 개수**:
  - 최소 3개

- **반드시 포함할 실패 시나리오**:
  - 선택된 칸이 바뀌어도 이전 하이라이트가 남는 경우
  - 상대 기물이나 빈 칸 선택 시 합법 수를 표시하는 경우
