## 0. 현재 코드 상태와 이 작업의 위치

- **현재 상태 요약**: 기물들의 일반 이동 로직은 완성되었습니다. 킹과 룩이 동시에 움직이는 특수 규칙인 캐슬링 로직이 필요합니다.
- **이 작업의 책임**: 킹사이드 및 퀸사이드 캐슬링 가능 여부를 판정하고, 실행 시 두 기물을 동시에 이동시킵니다.

- **이번 작업에서 하지 않는 것**: `[TASK-013]` (앙파상)에 연결된 후속 책임은 이번 태스크에서 함께 닫지 않는다.

- **경계 메모**:
  - 이번 태스크는 캐슬링(Castling) 처리 범위만 닫고, 후속 태스크 또는 인접 Feature의 세부 구현은 여기서 함께 처리하지 않는다.

## 🎯 1. 작업 목표

- **최종 상태**: 캐슬링 권한(`castlingRights`), 기물 사이의 빈 공간, 킹의 경로 안전성을 모두 만족할 때만 캐슬링을 합법 수로 반환합니다.

- **이번 작업의 최소 결과물**:
  - `packages/shared/src/engines/castling-engine.ts`
  - `packages/shared/src/engines/castling-engine.spec.ts`
- **성공 기준 (AC)**:
  - 캐슬링의 4가지 주요 제약 조건이 모두 정확히 판정된다.
  - 캐슬링 실행 시 킹과 룩이 올바른 최종 위치로 동시에 이동된다.
  - 캐슬링 직후 해당 방향의 권리가 소멸됨을 확인했다.

## 🛠️ 3. 상세 기술 사양

- **체크리스트 (모든 조건 만족 필수)**:
  1. `state.castlingRights` 비트 확인 (킹/룩 이동 이력 없음).
  2. 킹과 룩 사이의 모든 칸이 `null` 상태임.
  3. 현재 킹이 체크 상태가 아님.
  4. 킹이 통과하는 칸과 도착하는 칸이 공격받고 있지 않음 (`isSquareAttacked`).
- **상태 업데이트**: 캐슬링 착수 시 킹은 2칸 이동, 룩은 킹을 건너뛰어 안쪽 칸으로 이동.
- **필수 describe/it 목록**:
  - describe: `getCastlingMoves`
    - it: `모든 조건 만족 시 킹사이드/퀸사이드 캐슬링 좌표를 반환해야 한다`
    - it: `킹이 체크 상태일 때는 캐슬링이 불가능해야 한다`
    - it: `킹이 지나가는 경로가 공격받고 있으면 캐슬링이 불가능해야 한다`
    - it: `사이에 다른 기물이 있으면 캐슬링이 불가능해야 한다`

- **핵심 조립/흐름 규칙**:
  - 캐슬링의 4가지 주요 제약 조건이 모두 정확히 판정된다.
  - 캐슬링 실행 시 킹과 룩이 올바른 최종 위치로 동시에 이동된다.
  - 캐슬링 직후 해당 방향의 권리가 소멸됨을 확인했다.

- **데이터 모델 해석**:
  - 캐슬링은 킹/룩 위치, 경로 비어 있음, 캐슬링 권리, 통과 칸 공격 여부를 함께 다룹니다.
  - 출력은 현재 상태에서 허용되는 캐슬링 목적지 또는 특수 수 정보입니다.

- **외부 의존성**:
  - shared 내부 `isSquareAttacked`, `GameState`
  - `vitest`

- **import/export 규칙**:
  - `../architecture/project-rules.md`의 named export, 상대 경로 최소화, `.js` 확장자 규칙을 따른다.

- **권장 네이밍**:
  - 공개 함수/타입 이름: 캐슬링(Castling) 처리 책임이 드러나는 이름
  - 내부 helper 이름: 역할이 바로 드러나는 동사형 또는 조합형 이름
  - 핵심 변수명: 상태와 대상이 분명한 이름
  - 피해야 할 이름: data, item, obj, temp

- **이름별 사용 의도와 적용 시점**:
  - `getCastlingMoves`는 현재 상태에서 가능한 캐슬링 수만 뽑을 때 사용합니다.
  - `canCastleKingside`, `canCastleQueenside`는 내부 분기 이름으로 적절합니다.

- **인수 이름 가이드**:
  - `state`, `color`

- **짧은 예시 골격**:

```ts
const castlingMoves = getCastlingMoves(state, color);
```

- **최소 테스트 개수**:
  - 최소 3개

- **반드시 포함할 실패 시나리오**:
  - 체크 상태인데도 캐슬링을 허용하는 경우
  - 통과 칸 공격 여부를 검사하지 않는 경우
