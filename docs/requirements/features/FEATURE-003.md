# FEATURE-003 기보 메타데이터 입력 및 초안 저장 관리

## 상위 Epic

- [EPIC-001 학습용 기보 저장 및 분석](../epics/EPIC-001.md)

## 브랜치명

- `feature/FEATURE-003-메타데이터-초안저장`

## 목적

- 기보에 필요한 학습용 부가 정보(코멘트, 평가 등)를 입력하고, 작성 중인 기보를 초안으로 안전하게 보존 및 확정한다.

## 동작 개요

- UI 기준은 `docs/ui/FEATURE-003-metadata-draft-save.svg`를 기본안으로 삼고, 탭 상태는 `docs/ui/FEATURE-003-tab-comment.svg`, `docs/ui/FEATURE-003-tab-evaluation.svg`, `docs/ui/FEATURE-003-tab-game-info.svg`를 따른다.
- `FEATURE-002`에서 만든 보드, 수순 목록, SAN 탭 구조는 유지하고, `FEATURE-003`에서는 코멘트, 평가, 기보 정보, 초안 저장 상태, 정식 저장 액션을 확장한다.
- 기보와 메타데이터를 클라이언트에 저장하기 위한 로컬 스토리지 데이터 스키마를 설계한다.
- 수순 목록의 각 반수마다 코멘트와 착수 평가 기호를 입력할 수 있다.
- 게임 결과(백 승, 흑 승, 무승부), 종료 사유, 대국 날짜를 기록할 수 있다.
- 입력 중인 모든 데이터(보드, 수순, 코멘트 등)는 브라우저 내에 초안으로 자동 저장된다.
- 새로고침 시 기존 초안 데이터를 복원하여 입력을 이어나갈 수 있다.
- `초안 폐기`는 현재 초안을 삭제하고, `새 기보`는 기존 초안이 있을 때 사용자 확인 후 새 빈 기보를 시작한다.
- 작성이 완료된 기보는 수동 저장을 통해 정식 기보로 확정된다.
- 저장소 관리 기능(`FEATURE-004`)이 구현되기 전까지 정식 저장은 시스템 기본 저장소를 대상으로 처리한다.
- 시스템 기본 저장소는 사용자에게 별도 선택 UI를 요구하지 않는 임시 전제이며, 저장소 생성/선택/삭제 흐름은 `FEATURE-004`에서 별도로 고정한다.
- 정식 저장 성공 후 상세 화면 구현 자체는 `FEATURE-005`에서 다루며, `FEATURE-003`에서는 저장 성공 결과와 후속 상세 진입에 필요한 식별자 계약까지만 고정한다.

## 메모

- 초안은 시스템 상 항상 1개만 유지된다.
- 평가 기호의 `없음`은 별도 저장값이 아니라 현재 반수의 평가 기호를 제거하는 액션이다.
- 새 기보의 대국 날짜 기본값은 오늘 날짜이며, 사용자가 날짜를 지우면 날짜 필드는 비워진다.
- `FEATURE-003`은 `FEATURE-004`보다 먼저 진행하므로, 저장소 선택 UI가 없어도 정식 저장을 검증할 수 있도록 시스템 기본 저장소 전제를 둔다.
- `FEATURE-003`은 `FEATURE-005`보다 먼저 진행하므로, 저장된 기보 상세 화면 구현은 이 Feature의 완료 조건으로 보지 않는다.
- 정식 저장 이후에는 기보 상세 화면으로 이동하는 흐름과 자연스럽게 연결되어야 한다.
