## 추천 커밋 메시지

- `feat: 저장소 보드 기존 기보 업데이트 저장 추가`

## 목적

- 저장소 보드에서 기존 기보를 수정한 뒤 새 기보로 저장하지 않고 기존 기보를 업데이트하는 흐름을 고정한다.

## 완료 기준

- 기존 기보를 연 상태에서 저장 버튼을 누르면 기존 `gameId` 대상 업데이트 요청이 실행된다.
- 새 기보 저장과 기존 기보 업데이트 저장이 명확히 구분된다.
- 수순, 메타데이터, 코멘트, 어노테이션의 보존/삭제 기준이 저장 요청에 반영된다.
- 저장소 컨텍스트의 `repositoryId`와 업데이트 대상 `gameId`가 함께 유지된다.

## 관련 메모

- `TASK-034`에서는 저장 버튼의 위치와 비활성 상태만 고정한다.
- 실제 업데이트 저장 API와 상태 동기화는 이 Task에서 구현한다.
- `TASK-103`의 정식 기보 수순 수정 규칙과 충돌하지 않게 설계한다.
