- [구현 지침서 (Phase 3)](../../tasks/TASK-097.md)

## 상위 Feature

- [FEATURE-002 기보 입력 보드 상호작용 및 수순 관리](../features/FEATURE-002.md)

## 추천 커밋 메시지

- `feat: 착수 후 게임 종료 입력 차단 추가`

## 목적

- 착수 후 shared 엔진의 게임 결과 판정을 웹 상호작용 상태에 반영한다.
- 체크메이트, 스테일메이트, 50수 규칙, 3회 반복, 기물 부족 무승부 이후 새 착수 입력이 이어지지 않게 한다.

## 완료 기준

- 착수 완료 후 현재 `GameState`와 반복 이력을 기준으로 게임 종료 여부를 계산한다.
- 종료 상태에서는 새 착수 입력이 막힌다.
- 진행 중 상태에서는 기존 선택, 착수, 프로모션 흐름이 유지된다.
- 되돌리기/다시하기로 종료 이전 시점으로 이동하면 입력 가능 여부가 현재 시점에 맞게 갱신된다.
- 종료 결과와 종료 사유의 사용자 표시, 수순 말미의 `1-0`, `0-1`, `1/2-1/2` 표기는 `TASK-017`에서 다룬다.

## 관련 메모

- 정식 저장용 결과/종료 사유 입력은 `TASK-026`에서 별도로 다룬다.
- 이 Task는 자동 판정 결과를 새 착수 가능 여부에 반영하는 범위만 다룬다.
- 전용 종료 배너/패널과 별도 UI 스케치는 만들지 않는다.
