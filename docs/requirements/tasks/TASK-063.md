- `feat: 기보 및 초안 저장을 위한 데이터 스키마 정의`

## 목적

- 개별 기보, 입력 중인 초안, 그리고 기보 메타데이터(코멘트, 평가, 기보 정보)를 브라우저와 API에서 공통으로 검증할 수 있는 데이터 구조로 확립한다.

## 완료 기준

- 정식 저장된 기보 객체 구조(ID, 시스템 기본 저장소 ID, 게임 메타데이터, 수순 목록 및 FEN 내역, 수순별 코멘트/평가 등)를 정의한다.
- 현재 입력 중인 초안(Draft) 객체 구조를 별도로 정의한다.
- TypeScript를 사용한다면 명확한 타입 인터페이스(Interface)로 스키마가 선언되어야 한다.
- 다중 저장소 생성/선택/삭제 모델은 `FEATURE-004`에서 확장한다.
