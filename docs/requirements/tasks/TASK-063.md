## 추천 커밋 메시지

- `feat: 저장소 및 기보 저장을 위한 로컬 데이터 스키마 정의`

## 목적

- 여러 개의 저장소, 각 저장소 내부의 여러 기보, 그리고 개별 기보의 메타데이터(코멘트, 어노테이션)를 브라우저에 체계적으로 저장하기 위한 데이터 구조를 확립한다.

## 완료 기준

- 저장소(Repository) 객체 구조 (ID, 이름, 생성일 등)를 정의한다.
- 정식 저장된 기보 객체 구조 (ID, 소속 저장소 ID, 게임 메타데이터, 수순 목록 및 FEN 내역, 수순별 코멘트 맵 등)를 정의한다.
- 현재 입력 중인 초안(Draft) 객체 구조를 별도로 정의한다.
- TypeScript를 사용한다면 명확한 타입 인터페이스(Interface)로 스키마가 선언되어야 한다.
