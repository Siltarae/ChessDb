- `feat: 기보 정식 저장 API 추가`

## 목적

- 프론트엔드가 정식 저장을 요청할 수 있도록 서버의 기보 생성 API 계약과 저장 동작을 고정한다.
- `TASK-025`의 프론트 정식 저장 확정 흐름이 호출할 백엔드 진입점을 준비한다.

## 완료 기준

- `POST /api/games` 요청을 shared DTO/Zod 계약으로 검증한다.
- 유효한 기보 저장 요청은 시스템 기본 저장소에 정식 기보로 저장된다.
- 저장 성공 응답에는 후속 상세 진입에 사용할 저장 기보 ID가 포함된다.
- 잘못된 payload는 저장하지 않고 400 계열 오류로 거절한다.

## 관련 메모

- `FEATURE-004` 전까지 저장 대상은 시스템 기본 저장소로 고정한다.
- 저장 성공 후 프론트엔드 초안 정리와 사용자 피드백은 `TASK-025`에서 다룬다.
