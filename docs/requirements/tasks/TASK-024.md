## 추천 커밋 메시지

- `feat: 기존 초안 폐기 확인 후 새 기보 시작 추가`

## 목적

- 이미 초안이 있는 상태에서 새 기보를 시작할 때의 처리 규칙을 고정한다.
- UI 기준은 `docs/ui/FEATURE-003-metadata-draft-save.svg`의 초안 폐기와 새 기보 액션을 따른다.

## 완료 기준

- 기존 초안이 남아 있을 때 새 기보 시작 전 사용자 확인이 필요하다.
- 사용자가 확인하면 기존 초안을 버리고 새 기보를 시작한다.
- 확인 없이 기존 초안을 덮어쓰지 않는다.
- `초안 폐기`는 현재 초안을 삭제하고, `새 기보`는 기존 초안이 있을 때 확인 후 새 빈 기보를 시작한다.

## 관련 메모

- 초안을 여러 개 유지하는 모델은 현재 범위 밖이다.
