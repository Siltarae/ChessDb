## 목적

- 수동 저장이 초안을 어떤 상태로 바꾸는지 고정한다.
- UI 기준은 `docs/ui/FEATURE-003-metadata-draft-save.svg`의 정식 저장 액션을 따른다.

## 완료 기준

- 수동 저장은 초안을 정식 기보로 확정하는 동작이다.
- 수동 저장은 새 기보를 임의로 추가 생성하는 동작이 아니다.
- 수동 저장 버튼은 `TASK-095`에서 제공한 `POST /api/games` API를 호출한다.
- `FEATURE-004` 구현 전까지 수동 저장은 시스템 기본 저장소를 대상으로 처리된다.
- `FEATURE-005` 구현 전까지는 실제 상세 화면이 아니라 저장 성공 결과와 후속 상세 진입에 필요한 식별자 계약까지만 고정한다.
- 정식 저장 후 상세 뷰로 이동하는 흐름은 `FEATURE-005`에서 실제 화면으로 연결한다.

## 관련 메모

- 자동 저장은 초안 상태를 위한 기능이다.
- 백엔드 저장 API 구현은 `TASK-095`에서 다룬다.
- 시스템 기본 저장소는 `FEATURE-003` 진행을 위한 임시 전제이며, 사용자 저장소 생성/선택 흐름은 `FEATURE-004`에서 다룬다.
- 저장된 기보 목록과 상세 조회 화면은 `FEATURE-005`에서 다룬다.
