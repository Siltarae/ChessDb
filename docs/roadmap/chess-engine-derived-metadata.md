# 체스 엔진 기반 기보 메타데이터 자동 반영

## 상태

- 초안
- 현재 범위 밖 후순위 계획

## 배경

- 현재 `TASK-025` 정식 저장은 사용자가 입력한 draft 메타데이터를 기준으로 저장 가능 여부를 판단한다.
- 그러나 체스 엔진은 이미 착수 후 게임 상태와 종료 상태를 계산할 수 있다.
- 이 정보를 저장 메타데이터에 연결하지 않아, 사용자가 결과와 종료 사유를 직접 선택해야 저장 버튼이 활성화된다.

## 문제

- 체크메이트, 스테일메이트, 무승부 같은 엔진 계산 결과가 있어도 `metadata.result`, `metadata.terminationReason`에 자동 반영되지 않는다.
- 사용자는 보드상으로 게임이 끝난 것을 볼 수 있어도 정식 저장 조건을 별도로 채워야 한다.
- UI 입장에서는 저장 버튼이 비활성인 이유가 불명확해 보일 수 있다.

## 목표

- 체스 엔진이 계산한 게임 종료 정보를 기보 메타데이터 초깃값 또는 추천값으로 활용한다.
- 명확한 종료 상태는 `metadata.result`, `metadata.terminationReason`에 자동 반영할 수 있게 한다.
- 사용자가 수동으로 메타데이터를 수정한 경우 자동 반영이 덮어쓰지 않도록 한다.

## 범위 후보

- 현재 `gameState`와 `repetitionHistory`로 계산한 게임 결과를 draft metadata에 연결한다.
- 체크메이트, 스테일메이트, 반복 무승부, 기물 부족 같은 엔진 판정 결과를 저장 DTO의 결과/종료 사유 값으로 매핑한다.
- 자동 반영값과 사용자 수동 입력값의 우선순위 규칙을 정의한다.
- 저장 버튼 disabled 사유를 UI에서 더 명확히 표시한다.
- 자동 반영 후에도 사용자가 결과/종료 사유를 수정할 수 있는지 정책을 정한다.

## 제외 범위

- 현재 `TASK-025` 정식 저장 확정 구현
- 저장소 선택 UI
- 상세 화면 이동
- PGN import/export
- 엔진 규칙 자체의 추가 구현

## 승격 조건

- 정식 저장 UX에서 결과/종료 사유 수동 입력이 반복적으로 불편해질 때
- 게임 종료 감지와 메타데이터 입력의 중복이 실제 사용 흐름을 막을 때
- 저장 버튼 비활성 사유 안내만으로는 충분하지 않다고 판단될 때
