# UI 스케치 산출물 규칙

`docs/ui/`는 구현 전에 UI 방향을 빠르게 확인하기 위한 SVG 스케치 산출물을 관리한다.

## 목적

- 요구사항이나 Task를 실제 화면 구조로 검토한다.
- 레이아웃, 정보 계층, 주요 상태, 컴포넌트 배치를 구현 전에 합의한다.
- 사람이 Phase 5 구현에 들어가기 전에 시각 기준을 확인할 수 있게 한다.

## 파일 규칙

- Task 기반 스케치: `TASK-000-짧은-설명.svg`
- Task가 없는 탐색 스케치: `YYYY-MM-DD-짧은-설명.svg`
- 한 Task에서 상태가 여러 개면 파일을 분리하거나 하나의 SVG 안에 상태별 프레임을 나란히 둔다.

## SVG 작성 기준

- 스케치 시작 전에 와이어프레임 수준인지, 중간 충실도 mockup인지 먼저 정한다.
- `title`과 `desc`를 포함한다.
- 외부 이미지 링크를 사용하지 않는다.
- 실제 화면에 들어갈 수 있는 한국어 문구를 우선 사용한다.
- 색상, 간격, 컴포넌트 후보는 SVG 하단 또는 우측의 작은 메모 영역에 기록한다.
- 최종 구현 코드는 이 폴더에 작성하지 않는다.

## 사용 시점

- Phase 1에서 UI 요구사항이 모호할 때
- Phase 2에서 화면 구조나 디자인 패턴 선택이 필요할 때
- Phase 3에서 Task 지침서 작성 전에 화면 단위를 구체화할 때
- Phase 5 구현 중 사람이 시각 기준을 다시 확인하고 싶을 때
